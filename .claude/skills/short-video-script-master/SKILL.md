# 剧本创作大师

通过结构化对话，帮助用户从模糊的创意想法出发，逐步完成整部短剧的核心框架定调和故事创作。目标投放平台以红果短剧为主。

本 Skill 不调用任何外部 API，纯粹通过对话引导完成。

## 全局锚定规则

阶段一产出的 `framework.md` 是整部剧的**宪法文件**，贯穿所有后续阶段。在阶段二至阶段六的每一个步骤中，**必须始终读取并遵循** `framework.md` 中的以下内容：

- **核心人物形象**：角色外观、性格、动机、人物弧线 — 所有剧本对白、画面描述、关键帧 prompt 都必须与形象描述一致
- **角色关系**：关系类型和核心矛盾 — 所有互动和冲突必须符合已建立的关系设定
- **漫剧画风设定**：Style Lock（风格前缀）— 所有生图 prompt 必须携带风格前缀
- **整剧基调**：核心情绪和节奏特征 — 所有创作必须服务于已定的基调方向
- **付费节奏预埋**：前3集钩子和付费转化点 — 剧本创作时必须围绕这些节点设计

如果后续阶段发现 framework.md 中的设定与实际创作产生矛盾，应**暂停当前步骤**，与用户确认是否需要修改框架。框架一旦确认，后续所有内容都是框架的展开和落实，不得偏离。

## 工作流程

### 阶段一：创意收集与核心框架

通过开放式创意入口 + 热门灵感板激发创意，再逐步确认基础参数（画风 → 频向 → 规格 → 类型），通过深度对话收集核心素材，最终一次性生成完整的核心框架（剧情、人物、场景、画风、基调、差异化、付费钩子）。查阅 `reference/step1_ideation_and_framework.md` 获取完整的对话引导策略。当用户提到具体类型时，查阅 `reference/hongguo_genre_map.md` 向用户展示类型特征和爆款公式。当 6 个维度（画风、频向、规格、类型、核心情绪、一句话灵感）全部明确后，生成框架。**必须等待用户确认**才能进入阶段二。

### 阶段二：故事线设计

分两步完成。查阅 `reference/step2_story.md` 获取完整规范。

**2A — 大纲确认**：生成主线概述 + 所有篇章标题和一句话概览（含集数分配），**必须等待用户确认**才能进入 2B。

**2B — 篇章创作**：基于确认后的大纲，逐个创作每个篇章的完整故事，输出到 `stories/` 目录。全部完成后生成主线总文件 `story.md`。

### 阶段三：剧本创作与片段拆分

按篇章逐步推进。查阅 `reference/step3_screenplay.md` 获取完整规范。

**3A — 剧本生成**：读取篇章故事文件，为该篇章所有集数生成剧本（场景列表、画面、旁白对白、声音设计、结尾钩子）。用户确认后进入 3B。

**3B — 片段拆分**：将每集剧本拆分为 6-8 秒的 AI 视频片段，用朴实平白的语言描述画面内容（禁止比喻、华丽辞藻、抽象概念）。用户确认后展示**篇章完成选项**：

```
篇章「[篇章名]」的剧本和片段拆分已完成。请选择下一步：

1. 继续下一个篇章的剧本创作（→ STEP 3A）
2. 为本篇章生成生产脚本（→ STEP 4）
3. 返回上一个步骤修改
```

> 用户输入数字或描述选择。如果是最后一个篇章且用户选择1，提示所有篇章剧本已完成。

### 阶段四：生产脚本

将片段拆分转化为可直接用于 AI 视频生成的结构化生产指令。查阅 `reference/step4_production_script.md` 获取完整规范。

**4A — 全局参数提取**：从 framework.md 提取 Style Lock（风格前缀）和 Character Reference（角色外观关键词），与用户确认后进入 4B。

**4B — 分镜表生成**：为该篇章所有集数的每个片段生成结构化分镜表，每个片段拆分为 1-4 个分镜（镜号/时长/景别/运镜/画面描述/声音）。按篇章批量生成，用户确认后展示**篇章完成选项**：

```
篇章「[篇章名]」的生产脚本已完成。请选择下一步：

1. 继续下一个篇章的生产脚本（→ STEP 4B）
2. 为本篇章进入关键帧图片提示词（→ STEP 5）
3. 返回上一个步骤修改
```

> 用户输入数字或描述选择。如果是最后一个篇章且用户选择1，提示所有篇章生产脚本已完成。

### 阶段五：关键帧图片提示词

为 AI 图片生成准备结构化 prompt，通过预定义的角色和场景参考图提升跨片段一致性。查阅 `reference/step5_image_prompts.md` 获取完整规范。本阶段只输出 prompt，不调用外部 API。

**5A — 角色参考图 Prompt（项目级）**：从 framework.md 提取角色外观，为每个核心角色生成参考图 prompt（单张，默认穿搭，灰色纯色背景）。用户确认后进入 5B。

**5B — 场景参考图 Prompt（项目级）**：从 framework.md 提取场景描述，为每个场景生成参考图 prompt（无人物）。与 5A 一起向用户确认后进入 5C。

**5C — 关键帧 Prompt（篇章级）**：基于 STEP 4 分镜表的画面描述和 STEP 3B 的片段拆分，为该篇章所有集数的每个分镜生成首帧和末帧的完整图片 prompt，标注参考图引用。按篇章批量生成，用户确认后写入 `reference_images/` 目录。

### 阶段六：视频生成指令

基于实际生成的关键帧图片，为每个片段编写可直接用于 Seedance I2V 视频生成的结构化指令。查阅 `reference/step6_video_prompts.md` 获取完整规范。本阶段只输出指令文本，不调用外部 API。

**6A — 图片分析与片段组装（按集）**：扫描实际关键帧图片文件，确定每个片段的首帧/末帧图片来源，分析首帧/末帧的视觉差异（运镜、主体变化、环境变化），输出片段组装表。

**6B — 视频生成指令编写（按集）**：基于 6A 图片分析 + STEP 4 分镜表精炼，编写运镜公式、Motion Prompt、声音设计（对白/OS/环境音）、角色语音规格。按篇章批量生成，用户确认后写入 `video_generation/` 目录。
