# 阶段四：生产脚本

## 目标

将**STEP 3**产出的片段拆分（5-7s 中文叙事描述）转化为可直接用于 AI 视频生成的结构化分镜指令。每个片段（clip）拆分为 1-5 个分镜（sub-shot），以分镜表形式输出。

产出格式是通用的，不绑定具体供应商，为通用格式，可在任何图片/视频生成工具中使用。

## 背景：AI 视频生成的工作流

主流路径是**图生视频（I2V）**：
1. 用文生图模型生成首帧图片
2. 将首帧图片 + motion prompt 喂给视频生成模型
3. 部分模型支持原生音视频联合生成

STEP 4 只负责产出分镜表，不生成 prompt。图片 prompt 由 STEP 5 负责，视频 prompt 由 STEP 6 负责。

## 工作流程

### 前置校验（每次进入 STEP 4 时执行）

在开始生成之前，**必须**验证以下前置条件是否满足：

| 校验项 | 依赖来源 | 校验内容 | 不通过时处理 |
|--------|---------|---------|-------------|
| framework.md 存在 | STEP 1 | 文件存在且包含「核心人物」和「漫剧画风设定」字段 | 提示用户回到 STEP 1 生成核心框架 |
| 剧本文件存在 | STEP 3 | `screenplays/Chapter<序号>_<篇章名>/E<NN>.md` 对应该篇章所有集数都存在 | 提示用户回到 STEP 3 生成对应篇章的剧本 |
| 片段拆分完整 | STEP 3 | 每集剧本文件中包含「片段拆分」章节，且片段数量合理（每集约 6-12 个） | 提示用户回到 STEP 3 补充片段拆分 |

校验流程：
1. 用 Glob 工具扫描 `screenplays/Chapter<序号>_<篇章名>/` 目录下的 `E*.md` 文件，确认集数是否完整
2. 用 Grep 工具检查每集文件中是否包含 `## 片段拆分` 标题
3. 用 Read 工具读取 `framework.md`，确认包含「核心人物」和「漫剧画风设定」章节
4. 如果任何校验项不通过，向用户明确告知缺失内容和建议的回退步骤，**不要继续生成**

### 4A：全局参数提取

1. 用户选择要生成脚本的篇章/集数（或按顺序推进）
2. AI 读取以下文件：
   - `screenplays/Chapter<序号>_<篇章名>/E<NN>.md` — 该集的剧本 + 片段拆分（STEP 3 产出）
   - `framework.md` — 画风基调、核心人物形象、主要场景
3. 从 `framework.md` 中提取并生成：
   - **风格锁定**：全局风格前缀，中文，从「画风与风格」和「色调基调」提取而来
   - **角色参考**：每个角色的外观关键词，中文，从「核心人物」的「形象」字段提取而来
4. 与用户确认全局参数，确认后进入 4B

### 4B：分镜指令生成

为该篇章的所有集数的所有片段生成结构化分镜指令：

1. 逐集逐片段转化，每个片段（clip）拆分为 1-5 个分镜（sub-shot），以分镜表输出
2. 生成过程中执行**质量检查清单**（见下方）
3. 生成完毕后向用户展示该篇章全部集数的分镜表，**等待确认**
4. 确认后写入文件

**推翻机制**：
- 单个分镜不满意：修改该分镜
- 某集整体不满意：推翻重写该集
- 整个篇章不满意：推翻重来，从 4A 重新开始
- 风格锁定或角色参考需要调整：回到 4A 修改全局参数后重新生成

## 字段说明

### 分镜表列定义

| 列名 | 格式/内容 | 用途 |
|------|----------|------|
| 镜号 | `{片段号}-{分镜序号}`，如 `1-01`、`1-02` | 全局唯一标识 |
| 时长 | 秒数，如 `2s`、`1.5s` | 该分镜的持续时间 |
| 景别 | 大特写/特写/近景/中景/全景/远景/大远景 | 画面取景范围 |
| 运镜 | 运镜方式 + 可选幅度 | 摄像机运动 |
| 画面描述 | 分镜的首帧静态构图 + 运动变化 | 起始画面 + 运动变化的合并描述 |
| 声音 | 环境音/BGM/对白/OS | 声音设计 |

### Shot（景别）选项

| 景别 | 适用场景 |
|------|---------|
| 大特写 | 眼睛、嘴唇、手指等细节 |
| 特写 | 人物面部、手部动作 |
| 近景 | 胸部以上 |
| 中景 | 半身，腰部以上 |
| 全景 | 全身 |
| 远景 | 人物 + 周围环境 |
| 大远景 | 环境/城市场景 |

### Camera（运镜）选项

| 运镜 | 效果 | 分镜表写法 |
|------|------|-----------|
| 固定 | 稳定、观察 | `固定` |
| 推镜头 | 聚焦、强调情绪 | `缓慢推镜头` |
| 拉镜头 | 揭示环境 | `缓慢拉镜头` |
| 横摇 | 跟随水平运动 | `缓慢右摇` |
| 跟拍 | 跟随人物运动 | `移动跟拍` |
| 手持 | 纪实感、紧张感 | `轻微晃动` |
| 组合 | 景别+运镜同时变化 | `近景→特写，缓慢推镜头` |

运镜可组合景别变化：`近景→特写` 表示起幅近景、落幅特写。

### 画面描述规则

画面描述列合并了首帧构图和运动变化，必须严格遵循**语言平庸化**规则。这是给 AI 视频生成模型看的指令，不是文学作品。

### 结构

1. **首句描述起始画面**（静态）：
   - 用具体描述定义起始画面的人物姿态、环境状态、光线色调
   - 禁止动态动词（"正在走""正在说"）
2. **后续描述运动变化**（动态）：
   - 描述从起始画面开始的运动和变化
   - 如果有运镜，描述运镜带来的构图变化
3. **声音信息**（如有对白/OS）：
   - 对白格式：`[角色名]（[语气]）："[台词]"`
   - 内心OS格式：`[角色名]（内心OS）："[独白]"`

### 语言平庸化规则（全文件最严格的规则）

只描述肉眼可见的内容，每句话必须是一个可以直接看到的具体事实。使用短句、主谓宾结构。

**禁止清单**（以下表达一律禁止，没有例外）：
- **比喻/类比**：❌ "如同""宛如""像……一样""仿佛"
- **拟人/拟物**：❌ "风在低语""城市在沉睡""命运的齿轮转动""时间凝固"
- **成语/典故/诗词**：❌ "杀气腾腾""气吞山河""风云变色"
- **华丽形容词**：❌ "令人窒息的""摄人心魄的""惊心动魄的""震撼的""优雅的""孤傲的"
- **抽象概念**：❌ "气场""氛围""命运""宿命""绝望的深渊"
- **情绪化修饰**：❌ "狠狠地""冷冷地""微微一笑"→ 应改为"嘴角上扬"
- **主观评价**：❌ "帅气的""美丽的""恐怖的"→ 应改为具体的外形描述

**正确示范**：
```
男人坐在木床上，双手撑着床面，额头上可见汗珠，胸口快速起伏。房间里光线暗，左侧窗户透进来一道白色光线。男人把头转向右侧，看向房间角落。胸口起伏变慢，双手从床面上松开，放在膝盖上。
```

**错误示范**（绝对禁止）：
```
沈渡从简陋的木床上猛然坐起，满头冷汗，胸口剧烈起伏。晨曦透过破旧的窗棂洒入昏暗的偏房，那双深邃的眼眸中写满了惊恐与不甘。他缓缓环顾四周，目光如刀锋般锐利，仿佛在审视一个全然陌生的世界。
```

**自检方法**：写完每个分镜后，问自己——"这句话删掉后，读者还能知道画面里有什么吗？"如果能，说明这句话是废话或文学修饰，删掉它。

### 声音列规则

- **优先级**：对白/内心OS > 动作音效 > 环境音 > BGM
- **对白**：从 STEP 3 剧本中提取该分镜对应片段的对白
  - 格式：`[角色名]（语气）："[台词]"`
- **内心OS**：格式：`[角色名]（内心OS）："[独白]"`
- **环境音**：`雨声`、`风声`、`脚步声`、`心跳声` 等
- **BGM**：`紧张弦乐`、`安静`、`轻快钢琴` 等
- 多种声音用 `+` 连接：`喘息声+心跳声`
- 无声音的分镜写 `-`

## 分镜拆分原则

### 何时拆分

每个片段（5-7s）拆分为 1-5 个分镜，根据内容复杂度决定：

- **不拆分（1 个分镜）**：长镜头、单一运镜 + 单一主体变化，刻意营造沉浸感的片段
- **拆 2-3 个分镜**（最常见）：包含景别变化或情绪节奏转折的片段
- **拆 4-5 个分镜**：短剧节奏快，名场面和冲突爆发点需要密集的景别跳跃、主体切换来营造冲击力

> 短剧节奏极快，默认倾向使用更多分镜。仅在有明确艺术目的（如营造沉浸感、悬念停顿）时才使用长镜头。

### 拆分依据

| 情况 | 是否必须拆分 | 说明 |
|------|-------------|------|
| 景别变化 | 是 | 近景→特写、中景→全景等 |
| 主体切换 | 是 | 从人物切到物体、从一个角色切到另一个 |
| 情绪节奏转折 | 是 | 从安静到突然的动作 |
| 运镜方式切换 | 是 | 从固定到推镜头、从跟拍到静止 |
| 单一运镜+单一主体 | 否 | 保持 1 个分镜 |

### 时长规则

- 每个分镜 >= 1 秒
- 所有分镜时长之和 = 该片段时长
- 名场面/高潮片段的分镜可以更短（1-1.5s），制造紧凑节奏感
- 过渡/铺垫片段的分镜可以更长（3-4s），给观众喘息

### 连贯性规则

- 同一 clip 内，分镜 N 的末帧 = 分镜 N+1 的首帧（画面描述必须无缝衔接）
- 同一 clip 内最后一个分镜的末帧 = 下一个 clip 的第一个分镜的首帧（跨场景转场除外）
- 画面描述的首句必须为上一个分镜末尾状态的延续

## 创作要求

### 风格锁定生成规则

从 `framework.md` 的「画风与风格」和「色调基调」提取中文风格前缀：

- **画面风格**直接提取
- **色调基调**提取光线和色彩描述
- **视觉参考**提取核心关键词
- 末尾固定追加：「胶片颗粒感，电影级光影」

### 角色参考生成规则

从 `framework.md` 的「核心人物」的「形象」字段提取中文关键词：

- 保留所有具体的外貌描述（身高、体型、五官、穿着、配饰）
- 去掉主观评价（如"气质出众""眼神冷漠但笑起来很暖"）
- 格式为逗号分隔的关键词组

### 人物一致性

- 同一角色在同一篇章内的穿着应保持一致（除非剧情明确换装）
- 角色外观关键词必须与角色参考表一致

### 首尾帧人物锚定

- 如果一个片段中有重要角色出现，该角色至少在第一个或最后一个分镜中有体现
- 例外：纯环境/转场片段（无人物）、背景群演/路人

## 质量检查清单

每个片段生成后必须自检以下 10 项：

1. **首帧是静态画面？** — 每个分镜画面描述的首句没有"正在走""正在说"等动态动词
2. **运动描述了变化？** — 分镜画面描述中有明确的运动/变化
3. **角色外观一致？** — 画面描述中的角色关键词与角色参考表匹配
4. **首尾帧人物锚定？** — 片段中的重要角色至少在首/末分镜有外观体现
5. **运镜列完整？** — 景别和运镜信息明确
6. **风格一致？** — 画面描述中的风格与风格锁定一致
7. **连贯性？** — 同 clip 内相邻分镜画面无缝衔接；跨 clip 衔接（同场景）
8. **时长匹配？** — 所有分镜时长之和 = 该片段时长
9. **语言平庸化？** — 画面描述中没有比喻、拟人、成语、华丽形容词、抽象概念、主观评价。每句话都是肉眼可见的具体事实
10. **无废话？** — 删除任何不影响画面还原的句子后，逐句确认每句话都承载了不可省略的视觉信息

## 文件输出

### 目录结构

所有路径基于**项目根目录**，`production_scripts/` 与 `screenplays/` 同级：

```
outputs/dramas/<标题>_<yyyyMMdd_HHmm>/
  framework.md
  story.md
  stories/
    EP01_EP05_nightmare.md
  screenplays/
    Chapter1_噩梦/
      E01.md              # STEP 3 产出（剧本 + 片段拆分）
      E02.md
  production_scripts/                    # STEP 4 产出
    Chapter1_噩梦/
      E01.md              # 生产脚本（分镜表）
      E02.md
      ...
```

- 按篇章分子目录，目录名格式为 `Chapter<篇章序号>_<篇章中文名>`（与 `screenplays/` 中的篇章子目录一致）
- 每集一个文件 `E<NN>.md`，文件名与 screenplays 中的文件名对应
- 全局参数中的画面比例默认为 `9:16`（红果竖屏规格），分辨率默认为 `720p`（除非用户指定更高分辨率）

## 篇章完成选项（4B 确认后）

每个篇章的 4B 分镜指令生成确认并写入文件后，**必须**向用户展示互动选项，让用户决定下一步：

```markdown
篇章「[篇章名]」的生产脚本已保存到 `outputs/dramas/<标题>_<yyyyMMdd_HHmm>/production_scripts/Chapter<序号>_<篇章中文名>/`。请选择下一步：

1. 继续下一个篇章的生产脚本（→ STEP 4B）
2. 为本篇章进入关键帧图片提示词（→ STEP 5）
3. 返回上一个步骤修改
```

### 选项处理规则

- **选项1**：进入下一个篇章的 4B（如果全局参数 4A 未确认过则先走 4A）。如果是最后一个篇章，提示"所有篇章生产脚本已完成"并自动展示选项2。
- **选项2**：直接跳转到 STEP 5，为当前已完成的篇章生成关键帧图片提示词。
- **选项3**：回退到当前篇章的 4A 修改全局参数，或回退到上一个篇章。
- 用户也可以直接输入描述（如"进入STEP6""继续生成"等），AI 根据意图匹配选项。
- 支持**数字快捷输入**（直接输入 `1` `2` `3`）。

### 跨阶段跳转规则

当用户在 STEP 4 的篇章完成选项中选择"进入 STEP 5"后：
- STEP 5 完成后，用户可以选择继续 STEP 5 的下一个篇章，或回到 STEP 4 继续生成剩余篇章的生产脚本
- 这样用户可以先完整生成所有篇章的生产脚本，再统一进入 STEP 5；也可以逐个篇章完成生产脚本+关键帧

### 与 STEP 3 的衔接

如果用户是从 STEP 3 的篇章完成选项跳转到 STEP 4 的（而非顺序推进），则：
- STEP 4 当前篇章完成后，选项中额外增加一项：`4. 回到 STEP 3 继续下一个篇章的剧本创作`
- 确保用户可以在两个阶段之间灵活跳转，不必严格按顺序逐阶段完成
