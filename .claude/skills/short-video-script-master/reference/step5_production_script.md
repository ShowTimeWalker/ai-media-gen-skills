# 阶段五：生产脚本

## 目标

将**STEP 4B**产出的片段拆分（6-8s 中文叙事描述）转化为可直接用于 AI 视频生成的结构化生产指令。每个片段独立产出首帧图 prompt + 视频 motion prompt。

产出格式是通用的，不绑定具体供应商，由 content-generation-workflow 在实际调用时适配。

## 背景：AI 视频生成的工作流

主流路径是**图生视频（I2V）**：
1. 用文生图模型生成首帧图片（如 Seedream）
2. 将首帧图片 + motion prompt 喂给视频生成模型（如 Seedance）
3. 部分模型（如 Seedance 1.5 Pro）支持原生音视频联合生成

因此每个片段需要产出：**一张首帧图 prompt + 一段视频 motion prompt**。

## 工作流程

### 5A：全局参数提取

1. 用户选择要生成脚本的篇章/集数（或按顺序推进）
2. AI 读取以下文件：
   - `screenplays/<篇章缩写>/E<NN>.md` — 该集的剧本 + 片段拆分（STEP 4B 产出）
   - `framework.md` — 画风基调、核心人物形象、主要场景
3. 从 `framework.md` 中提取并生成：
   - **风格锁定**：全局风格前缀，中文，从「画风与风格」和「色调基调」提取而来
   - **角色参考**：每个角色的外观关键词，中文，从「核心人物」的「形象」字段提取而来
4. 与用户确认全局参数，确认后进入 5B

### 5B：片段指令生成

为该篇章的所有集数的所有片段生成结构化生产指令：

1. 逐集逐片段转化，每个片段产出：景别 / 角度 / 运镜 / 首帧图 Prompt / 视频 Motion Prompt / 末帧
2. 生成过程中执行**质量检查清单**（见下方）
3. 生成完毕后向用户展示该篇章全部集数的生产指令，**等待确认**
4. 确认后写入文件

**推翻机制**：
- 单个片段 prompt 不满意：修改该片段
- 某集整体不满意：推翻重写该集
- 整个篇章不满意：推翻重来，从 5A 重新开始
- 风格锁定或角色参考需要调整：回到 5A 修改全局参数后重新生成

## 产出格式

每集一个文件，Markdown 格式：

```markdown
# E<NN> Production Script

> 集数：第[X]集 | 篇章：[篇章名] | 时长：[X]秒 | 片段数：[X]

## 全局参数

| 参数 | 值 |
|------|------|
| 画面比例 | 9:16 |
| 分辨率 | 480p |

## 风格锁定

[中文风格前缀。从 framework.md 的画风与风格 + 色调基调提取而来]

## 角色参考

| 角色 | 外观关键词 |
|------|-----------|
| [角色名] | [中文外观关键词] |
| [角色名] | [中文外观关键词] |

## 片段列表

### C01（[X]秒）

**景别**：[景别] | **角度**：[角度] | **运镜**：[运镜]

**出场角色**：[角色列表]

**首帧图 Prompt**：

[中文，描述片段起始画面的静态构图]

**视频 Motion Prompt**：

[中文，描述 6-8s 内的运动和变化]

**末帧**：

[中文，1-2 句。末帧画面描述，用于连贯性校验]

---
```

## 字段说明

| 字段 | 字数/格式 | 用途 |
|------|----------|------|
| Shot（景别） | 见下表 | 画面取景范围 |
| Angle（角度） | 见下表 | 摄像机角度 |
| Camera（运镜） | 见下表 | 摄像机运动方式 |
| Characters | 逗号分隔 | 该片段出场的角色 |
| 首帧图 Prompt | 描述起始画面的静态构图 | 文生图 prompt |
| 视频 Motion Prompt | 描述运动和变化 | 图生视频 prompt |
| 末帧 | 1-2 句 | 末帧描述，连贯性校验用 |

### Shot（景别）选项

| 景别 | 适用场景 |
|------|---------|
| 大特写 | 眼睛、嘴唇、手指等细节 |
| 特写 | 人物面部、手部动作 |
| 近景 | 胸部以上 |
| 中景 | 半身，腰部以上 |
| 全景 | 全身 |
| 远景 | 人物 + 周围环境 |
| 大远景 | 环境/城市场景 |

### Angle（角度）选项

| 角度 | 效果 |
|------|------|
| 平视 | 客观中性 |
| 仰视 | 人物显得高大/有压迫感 |
| 俯视 | 人物显得渺小/弱势 |
| 鸟瞰 | 上帝视角，展示空间关系 |
| 倾斜 | 不安、紧张、失衡感 |

### Camera（运镜）选项

| 运镜 | 效果 |
|------|------|
| 静止 | 稳定、观察 |
| 推镜头 | 聚焦、强调情绪 |
| 拉镜头 | 揭示环境、疏离感 |
| 左摇 / 右摇 | 跟随水平运动 |
| 上摇 / 下摇 | 跟随垂直运动或揭示 |
| 移动跟拍 | 跟随人物运动 |
| 环绕 | 环绕主体，360度展示 |
| 升降 | 视角高度变化 |
| 手持 | 纪实感、紧张感 |

运镜可组合，如：`静止 → 缓慢推镜头` 或 `移动跟拍 → 环绕`。

## 创作要求

### 风格锁定生成规则

从 `framework.md` 的「画风与风格」和「色调基调」提取中文风格前缀：

- **画面风格**直接提取：如「现代都市写实，偏电影质感」
- **色调基调**提取光线和色彩描述：如「前半段暗调冷灰」→「冷色调，低饱和，暗调」
- **视觉参考**提取核心关键词：如「类似《隐入尘烟》的写实底色」→「写实风格，有胶片质感」
- 末尾固定追加：「胶片颗粒感，电影级光影」

### 角色参考生成规则

从 `framework.md` 的「核心人物」的「形象」字段提取中文关键词：

- 保留所有具体的外貌描述（身高、体型、五官、穿着、配饰）
- 去掉主观评价（如"气质出众""眼神冷漠但笑起来很暖"）
- 格式为逗号分隔的关键词组，方便直接嵌入首帧图 Prompt

**示例**：
- 原文：「身高183cm，偏瘦但肩膀宽，五官棱角分明，眉骨高、眼窝略深，薄唇。常穿深色衬衫卷袖子到小臂，左腕戴一块旧款机械手表，头发微长偶尔有一缕垂到额前」
- 转化：`男性，28岁，身高183cm，偏瘦但肩膀宽，五官棱角分明，眉骨高、眼窝略深，薄唇，深色衬衫袖子卷到小臂，左腕旧款机械手表，头发微长有一缕垂到额前`

### 首帧图 Prompt 规则

- **描述的是静态画面**，不是动态过程
- 必须覆盖：人物（外貌、穿着、表情、姿态）、环境（场景、光线、色调）、构图
- 角色外观关键词必须与角色参考表完全一致
- 风格词必须与风格锁定一致
- 遵循 Seedream 规范：`主体 + 姿态 + 环境 + 风格/光影/构图`
- **禁止动态描述**：
  - ❌ "一个男人正在走廊里走"
  - ✅ "一个男人的右脚迈出一步，站在昏暗走廊中间"
  - ❌ "她正在哭"
  - ✅ "她的眼角有泪水滑下，眼睛发红"
- **禁止比喻和抽象概念**（继承 STEP 4B 的语言规则）

### 视频 Motion Prompt 规则

- 描述从首帧开始的 6-8 秒内会发生什么**变化**
- 遵循 Seedance 规范：`主体 + 运动 + 环境 + 运镜/切镜 + 美学描述`
- 用程度副词控制节奏：缓慢地、轻轻地、快速地、用力地、突然地
- **运镜描述**使用 Seedance 的公式：`起幅构图 + 运镜 + 运镜幅度 + 落幅构图`
  - 示例：`中景起幅，缓慢推镜头至男人面部特写`
- **声音描述**（可选）：如果片段中有对白或内心OS，包含声音描述
  - 对白格式：`一个二十多岁的男性声音，语气压抑："……"`
  - 内心OS格式：`男性内心独白，低沉克制："……"`
  - 环境音：`窗外有雨声，远处有车流声`
- **不要重复首帧中已描述的静态信息**，视频 Motion Prompt 聚焦于变化和运动

### 连贯性规则

- C(N) 的末帧画面应该能与 C(N+1) 的首帧画面自然衔接
- **同场景内**相邻片段：首帧构图应与上片段末帧接近（同机位或自然过渡）
- **跨场景转场**：首帧为新场景的起始构图（如 C03 的闪白转场，首帧就是新场景的暖黄灯光）
- 生成时逐片段检查连贯性，如果不衔接需要在视频 Motion Prompt 或末帧中调整

### 首尾帧人物锚定

- 如果一个片段中有重要角色出现，该角色的外观至少要在首帧图或末帧中有体现（两者都有更好）
- 例外：纯环境/转场片段（无人物）、背景群演/路人（非核心角色）
- 禁止的情况：角色只出现在片段中间，首帧和尾帧都没有该角色
- 如果发现 STEP 4B 的片段拆分违反了此规则，需要回退到 STEP 4B 重新拆分

### 人物一致性

- 每个片段中出现的角色，其首帧图 Prompt 中的外观描述必须与角色参考表一致
- 同一角色在同一篇章内的穿着应保持一致（除非剧情明确换装）
- 角色在不同片段中的表情、姿态可以不同，但五官、体型、穿着特征不可变

## 质量检查清单

每个片段生成后必须自检以下 8 项：

1. **首帧是静态画面？** — 首帧图 Prompt 中没有"正在走""正在说"等动态动词，只有姿态和状态描述
2. **Motion 描述了变化？** — 视频 Motion Prompt 中有明确的运动/变化，不是对首帧画面的重复描述
3. **角色外观一致？** — 首帧图 Prompt 中的角色关键词与角色参考表匹配
4. **首尾帧人物锚定？** — 片段中出现的重要角色，至少在首帧图或末帧中有外观体现（禁止只在中间出现）
5. **运镜字段一致？** — 视频 Motion Prompt 中的运镜描述与景别/角度/运镜字段一致
6. **风格一致？** — 首帧和视频 prompt 中的风格词与风格锁定一致
7. **连贯性？** — 该片段的末帧与下一个片段的首帧能自然衔接
8. **时长匹配？** — 视频 Motion Prompt 中描述的运动量合理匹配 6-8 秒时长

## 文件输出

### 目录结构

所有路径基于**项目根目录**，`production_scripts/` 与 `screenplays/` 同级：

```
outputs/scripts/<标题>_<yyyyMMdd>/
  idea.md
  framework.md
  story.md
  stories/
    nightmare_1_5.md
  screenplays/
    nightmare/
      E01.md              # STEP 4 产出（剧本 + 片段拆分）
      E02.md
  production_scripts/                    # STEP 5 产出
    nightmare/
      E01.md              # 生产脚本（首帧 + motion prompt）
      E02.md
      ...
```

- 按篇章分子目录，目录名与 `screenplays/` 中的篇章子目录一致
- 每集一个文件 `E<NN>.md`，文件名与 screenplays 中的文件名对应
- 全局参数中的画面比例默认为 `9:16`（红果竖屏规格），分辨率默认为 `480p`（除非用户指定更高分辨率）

告知用户："篇章「[篇章名]」的生产脚本已保存到 `outputs/scripts/<标题>_<yyyyMMdd>/production_scripts/<篇章缩写>/`。是否继续下一集？"
