# 阶段一：创意收集

## 目标

通过自由对话收集足够的创作素材，为生成故事概览做准备。

## 开场白

开场时分三步引导用户确认基础参数：**画风 → 规格 → 类型**。先查阅 `reference/hongguo_genre_map.md` 获取最新类型数据，然后按以下格式输出：

### 第一步：画风选择

```markdown
你好，我是剧本创作大师 🎬

在开始之前，先确认一下你想要什么画风？输入编号选择：

**🎨 画风选择**

1. 🏮 **国潮古风** — 色彩浓郁、东方古典韵味（适合：重生复仇、穿书甜宠、权谋、种田）
2. 🌸 **日系二次元** — 大眼睛精致面部、色彩明亮柔和（适合：校园恋爱、穿书甜宠、闪婚、双女主、萌宝）
3. 🇰🇷 **韩漫精致风** — 高品质上色、人物造型写实唯美（适合：甜宠霸总、虐恋追妻、大女主、闪婚）
4. 🖤 **暗黑哥特风** — 低饱和度、强对比光影（适合：复仇悬疑、重生复仇、虐恋、末世）
5. 🌿 **治愈清新风** — 低饱和度暖色调、水彩质感（适合：治愈救赎、种田生活流、萌宝甜宠、先婚后爱）

> 所有项目均为 AI 漫剧形式，主角年龄 18-24 岁。
> 也可以直接告诉我你的想法。
```

等待用户选择画风后，进入第二步。

```markdown
**中等体量（50-70分钟）**
14. 20集 × 3分钟 = 60分钟
15. 30集 × 2分钟 = 60分钟

**大体量（75-120分钟）**
16. 25集 × 3分钟 = 75分钟
17. 30集 × 3分钟 = 90分钟
```

```markdown

**🔥 男频推荐**（根据体量筛选，见下方分级规则）
[...带编号的类型列表，编号 13-16...]
```

### 体量分级推荐规则

根据用户选择的规格，在开场白中只推荐适合该体量的类型，避免小体量推荐复杂叙事：

| 体量 | 对应选项 | 适用特征 | 推荐类型 |
|------|---------|---------|---------|
| **试水级（5-15分钟）** | 1-3 | 单线叙事、场景少、人物少（2-3人）、一个核心爽点打透 | 穿书甜宠（单集爽点型）、甜宠霸总（甜宠日常片段）、萌宝甜宠（萌娃名场面集锦） |
| **微短级（16-25分钟）** | 4-8 | 单线叙事、可有小反转、人物 3-4 人 | 穿书甜宠、闪婚先婚后爱、甜宠霸总、种田生活流、治愈救赎、大女主（简单逆袭线） |
| **短剧级（26-45分钟）** | 9-13 | 单线+辅线、可有一个小篇章反转、人物 4-5 人 | 以上全部 + 重生复仇（单线速通）、真假千金、双女主、虐恋追妻（精简版） |
| **中等体量（50-70分钟）** | 14-15 | 多线并行、完整故事弧、人物 5-7 人、可分 2-3 篇章 | 全部女频类型 + 全部男频类型 |
| **大体量（75-120分钟）** | 16-19 | 多线深度、完整世界观、人物 7-10 人、多篇章叙事 | 全部类型，可叠加多重设定（系统+种田、穿书+真假千金等融合类型） |
| **超大体量（150-180分钟）** | 20-21 | 复杂多线、完整世界观、人物 7-10 人、多篇章深度叙事 | 全部类型，可叠加多重设定（系统+种田、穿书+真假千金等融合类型） |

## 对话引导策略

接受任何形式的输入：关键词、一段话、一个情绪、一种画面感、甚至"我也不知道"。

在自由对话中，根据用户的输入自然地追问以下关键维度（不要像填表一样逐个问，而是融入对话）：

| 维度 | 引导问题 | 为什么重要 |
|------|---------|-----------|
| 画风风格 | "你喜欢什么画风？古风、日系、韩漫？" | 已在开场第一步确认，决定后续所有 AI 生图的视觉基底 |
| 剧集规格 | "你想做多大规模？试水还是正式上线？" | 已在开场第二步确认，决定类型推荐和故事复杂度 |
| 类型方向 | "你想要甜一点、虐一点、还是爽一点的？" | 已在开场第三步确认，决定整体调性和目标受众 |
| 频向 | "你想做男频还是女频？" | 男频女频的爽点机制完全不同 |
| 核心情绪 | "观众看完应该是什么感觉？爽？甜？燃？还是又哭又笑？" | 情绪是短剧的灵魂 |
| 一句话灵感 | "如果只能用一句话讲这个故事，你会怎么说？" | 检验故事核是否足够清晰有力 |
| 参考作品 | "有没有你特别喜欢的短剧或电影想参考？" | 帮助快速对齐审美和期望 |

## 类型引导

当用户提到某个类型关键词或输入编号时，查阅 `reference/hongguo_genre_map.md`，向用户展示该类型的特征、爆款公式和代表作，帮助用户确认方向。

### 编号解析规则

当用户输入类型编号时，按以下规则处理：
- **单个编号**（如「5」）：展示该类型的完整特征，然后继续引导核心情绪和灵感
- **组合编号**（如「1+13」「5+19」）：展示各类型的特征，然后引导用户确定主类型和融合方式
- **编号+补充**（如「5，但不要太爽」）：以编号类型为基础，结合用户的补充需求调整方向

## 退出条件与文件输出

当对话中已覆盖以下核心维度时，AI 主动告知用户信息已足够：

1. **画风风格**已明确（开场第一步已确认）
2. **剧集规格**已明确（开场第二步已确认）
3. **类型方向**已明确（开场第三步已确认）
4. **核心情绪**已明确（知道想让观众感受什么）
5. **一句话灵感**已明确（能说出故事的核）

### 输出文件

退出前必须将收集到的创意整理并写入文件：

**文件路径**：项目根目录下的 `outputs/scripts/<标题>_<yyyyMMdd>/idea.md`（注意：是项目根目录的 outputs，不是 skill 目录下的）

**标题规则**：
- 由 AI 根据创意核心提炼，简洁有力，2-6 个字
- 使用英文命名（如 `reborn_chess`、`book_transmigrate`、`tycoon_rise`）
- 如果是中文标题，用拼音下划线格式（如 `chong_sheng_qi_ju`）

**idea.md 格式**：

## 常见引导话术

| 用户输入 | 引导策略 |
|---------|---------|
| "随便来一个" | 推荐当前红果最火的2-3个类型，让用户选 |
| "我想做一个甜甜的" | 确认频向后，展示甜宠类子类型（穿书甜宠/闪婚/先婚后爱） |
| "我想做战神" | 查阅类型图谱，展示战神类爆款公式和代表作，引导细化设定 |
| 只给情绪词"爽" | 询问想通过什么方式爽——逆袭翻盘？碾压打脸？智斗反转？ |
| "我想做一个跟XX剧差不多的" | 分析参考作品的核心类型和爽点，在此基础上引导差异化 |
| "我不知道做什么" | 用开放式问题引导："你最近有没有什么画面或情绪特别打动你？" |
