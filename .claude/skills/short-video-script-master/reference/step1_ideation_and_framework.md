# 阶段一：创意收集与核心框架

## 目标

通过对话引导，从用户的创意想法出发，确认基础参数、深化创意素材，最终一次性生成完整的核心框架定调。

本阶段替代原有的「创意收集」+「核心框架定调」两个阶段，合并为一步完成，减少中间环节和重复确认。

---

## 第一部分：创意入口

### 开场白

```markdown
你好，我是剧本创作大师 🎬

在聊具体的之前——你现在脑子里有没有一个画面、一句话、或者一种感觉？

比如"我想做一个重生复仇的""我想要甜甜的""我看到了一个画面……"

什么都行，哪怕只有两个字。如果还没想好，输入 0 我给你看看最近什么最火。

> 🧪 快速测试？输入「测试模式」跳过繁琐流程，自动跑通全链路。
```

- 用户有想法 → 捕获创意火花，记录到上下文中，进入第二部分（热门灵感板）
- 用户输入 0 / 不知道 / 随便 → 直接进入第二部分（热门灵感板）
- 用户输入「测试模式」「测试」「test」→ **触发测试模式**（见下方）

### 测试模式触发

当用户在创意入口输入「测试模式」「测试」或「test」时：

1. **立即加载** `reference/test_mode.md`，按其中的规范执行
2. 向用户确认类型方向（仍需一次简短交互）：

```markdown
🧪 测试模式已激活！固定参数：480p / 30集 × 1分钟 / 9:16

只需告诉我：你想做什么类型？（一句话或编号都行）
比如"末世生存""甜宠""5"——其他的我都自动搞定。
```

3. 用户给出类型方向后，**AI 自主完成以下全部工作**（不再与用户互动）：
   - 自动匹配画风 → 生成 framework.md（固定参数覆盖）
   - 自动生成故事大纲 → 仅创作第一篇章故事
   - 自动生成第一集剧本 → 前半集片段拆分（后半集占位）
   - 自动生成分镜表（第一集前半集）
   - 自动生成图片提示词（≤5角色 + ≤8场景 + 关键帧）
   - 自动生成视频指令（第一集前半集）
   - 输出完成报告
4. 全程遵循 `reference/test_mode.md` 中的规则，**遇到冲突以测试模式设定为准**

### 创意入口引导话术

| 用户输入 | 引导策略 |
|---------|---------|
| 有明确想法（关键词/一句话/一个画面） | 肯定创意，追问"这个画面/感觉里，最打动你的是什么？"，然后进入灵感板 |
| 模糊情绪（"甜甜的""爽一点"） | 追问"你想通过什么方式爽/甜？逆袭翻盘？碾压打脸？日常撒糖？"，然后进入灵感板 |
| "随便来一个" | 直接进入灵感板，让 TOP5 激发灵感 |
| "我不知道做什么" | 直接进入灵感板 |
| 「测试模式」「测试」「test」 | 触发测试模式，加载 `reference/test_mode.md` |

## 第四部分：深度对话

基础参数确认后，通过自由对话深化创意素材。根据用户的输入自然地追问以下维度（不要像填表一样逐个问，而是融入对话）：

| 维度 | 引导问题 | 为什么重要 |
|------|---------|-----------|
| 核心情绪 | "观众看完应该是什么感觉？爽？甜？燃？还是又哭又笑？" | 情绪是短剧的灵魂 |
| 一句话灵感 | "如果只能用一句话讲这个故事，你会怎么说？" | 检验故事核是否足够清晰有力 |
| 参考作品 | "有没有你特别喜欢的短剧或电影想参考？" | 帮助快速对齐审美和期望 |
| 差异化钩子 | "你觉得你的故事跟同类型的有什么不一样？" | 红果同类型上百部，差异化决定生死 |

### 深度对话引导话术

| 用户输入 | 引导策略 |
|---------|---------|
| 只给情绪词"爽" | 询问想通过什么方式爽——逆袭翻盘？碾压打脸？智斗反转？ |
| "我想做一个跟XX剧差不多的" | 分析参考作品的核心类型和爽点，在此基础上引导差异化 |
| 给了完整故事 | 确认核心情绪和差异化，准备进入框架生成 |
| "随便/你看着来" | 基于已确认的类型和频向，AI 自行补全，直接进入框架生成 |

---

## 退出条件

当以下 6 个维度全部明确后，AI 主动告知用户信息已足够，直接生成框架：

1. **画风风格**已明确（第三部分参数一）
2. **频向**已明确（第三部分参数二）
3. **剧集规格**已明确（第三部分参数三）
4. **类型方向**已明确（第三部分参数四）
5. **核心情绪**已明确（第四部分深度对话）
6. **一句话灵感**已明确（第四部分深度对话）

如果用户在创意入口或灵感板中已给出足够信息，可以跳过部分深度对话，直接生成框架。

## 第五部分：框架生成与确认

### 画风细化

基于已确认的画风大类，根据题材和情绪方向细化以下维度：

**色调基调**（根据题材自动推荐，用户可调整）：
- 暖调柔光（甜宠治愈感） / 暗调冷色（权谋悬疑感） / 高对比强光（复仇热血感） / 低饱和度清新（种田生活流） / 赛博霓虹（末世科幻感）

**画面质感**（根据画风自动推荐）：
- 厚涂油画感 / 赛璐璐平涂 / 水彩手绘感 / 噪点肌理感

**角色比例**（默认推荐）：
- 日系/韩漫/国潮：7-8 头身（修长优美）
- 治愈/可爱：5-6 头身（Q 版可爱）

> 注意：红果 2026 新规要求禁止呈现画风观感不适的内容，暗黑风需注意尺度。

### 输出格式


### 确认话术

```
以上是整部剧的核心框架。请确认是否符合你的想法：

1. 确认，开始创作故事大纲
2. 调整某个方向（请告诉我你想改什么）
3. 推翻重来，换个方向
```

**必须等待用户确认**，才能进入阶段二（故事线设计）。

- 用户选择"调整"时，修改后重新展示框架并再次等待确认
- 用户选择"推翻重来"时，回到创意入口重新开始

## 文件输出

**文件路径**：项目根目录下的 `outputs/scripts/<标题>_<yyyyMMdd>/framework.md`

**标题规则**：
- 由 AI 根据创意核心提炼，简洁有力，2-6 个字
- 使用英文命名（如 `reborn_chess`、`book_transmigrate`、`tycoon_rise`）
- 如果是中文标题，用拼音下划线格式（如 `chong_sheng_qi_ju`）

**输出规则**：
- 将框架全部内容写入文件（不含末尾的确认选项部分）
- 同时在对话中告知用户："核心框架已保存到 `outputs/scripts/<标题>_<yyyyMMdd>/framework.md`（项目根目录下）。"
- 然后进入**阶段二**（故事线设计，查阅 `reference/step2_story.md`）

## 关键规则

- **所有项目固定为 AI 漫剧形式**，不需要向用户确认制作形式
- **主角年龄严格控制在 18-24 岁之间，不超过 24 岁**。配角可以有其他年龄段
- 核心人物的**形象描述必须具体**且符合漫剧风格，避免"长得好看""气质出众"等模糊描述
- 主要场景要标注出现频率，场景描述要包含漫剧画风的视觉质感
- 画风设定中必须包含 **AI 生图 Style Lock**，作为后续所有生图 prompt 的风格前缀
- 整剧基调要与核心情绪一致
- 剧情概要控制在 300-500 字，不能只写设定要有剧情走向
- 角色关系图谱中的每条关系都要有矛盾或张力，不能只是"认识"或"喜欢"
