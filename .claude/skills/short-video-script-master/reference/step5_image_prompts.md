# 阶段五：关键帧图片提示词

## 目标

在生成每个片段的首帧和末帧图片之前，先预生成角色参考图和场景参考图的 prompt，再基于参考图为每个片段的首帧和末帧编写图片 prompt，显著提升人物和场景的一致性。

产出格式是通用的，不绑定具体供应商，为通用格式，可在任何图片/视频生成工具中使用。

## 背景：为什么需要参考图

STEP 4 的分镜表提供了每个分镜的画面描述（包含静态起始画面和运动变化）。实际生图时，即使描述中包含详细的角色外观信息，AI 模型每次生成的角色外观仍会有差异。

通过预先生成角色参考图和场景参考图，后续关键帧生成时使用 image-to-image 模式（参考图作为输入），可以显著提升一致性。

本 STEP 只输出结构化 prompt，不调用外部 API。实际生图由后续工具执行。

## 工作流程

### 前置校验（每次进入 STEP 5 时执行）

在开始生成之前，**必须**验证以下前置条件是否满足：

| 校验项 | 依赖来源 | 校验内容 | 不通过时处理 |
|--------|---------|---------|-------------|
| framework.md 存在 | STEP 1 | 文件存在且包含「核心人物」形象字段和「主要场景」描述 | 提示用户回到 STEP 1 生成核心框架 |
| 生产脚本存在 | STEP 4 | `production_scripts/Chapter<序号>_<篇章名>/E<NN>.md` 对应该篇章所有集数都存在 | 提示用户回到 STEP 4 生成生产脚本 |
| 生产脚本内容完整 | STEP 4 | 每集文件中包含「分镜表」章节，且每个片段有完整的镜号/时长/景别/运镜/画面描述/声音 | 提示用户回到 STEP 4 补充完整生产脚本 |
| 剧本文件存在 | STEP 3 | `screenplays/Chapter<序号>_<篇章名>/E<NN>.md` 对应该篇章所有集数都存在 | 提示用户回到 STEP 3 生成剧本 |

校验流程：
1. 用 Read 工具读取 `framework.md`，确认包含「核心人物」中每个角色的「形象」字段和「主要场景」表格
2. 用 Glob 工具扫描 `production_scripts/Chapter<序号>_<篇章名>/` 目录，确认集数完整
3. 用 Grep 工具检查每集生产脚本中是否包含「分镜表」章节
4. 用 Glob 工具扫描 `screenplays/Chapter<序号>_<篇章名>/` 目录作为辅助参考
5. 如果任何校验项不通过，向用户明确告知缺失内容和建议的回退步骤，**不要继续生成**

### 5A：角色参考图 Prompt（按项目，首次执行）

1. 读取 `framework.md` 提取所有核心人物的「形象」字段
2. 读取当前篇章的剧本文件（`screenplays/Chapter<序号>_<篇章名>/E<NN>.md`），识别每个角色实际出现的穿搭
3. 从 `framework.md` 提取风格锁定（与 STEP 4 一致）
4. 为每个核心角色生成 1 张参考图 prompt：
   - **角色参考图**（单张，默认穿搭）：使用该角色在剧中最常出现的默认穿搭，景别为近景
5. 向用户展示全部角色的参考图 prompt，**等待确认**

### 5B：场景参考图 Prompt（按项目，首次执行）

1. 读取 `framework.md` 提取所有「主要场景」的描述
2. 为每个场景生成 **1 张单图 prompt**，选取该场景最具代表性的视角
3. 场景参考图**不包含人物**，纯环境
4. 向用户展示全部场景的参考图 prompt，**等待确认**

5A 和 5B 一起向用户展示确认。后续篇章如需新增角色或场景，补充生成对应参考图 prompt 即可，无需重新生成全部。

### 5C：关键帧 Prompt（按篇章）

1. 用户选择要生成关键帧的篇章（或按顺序推进）
2. 读取以下文件：
   - `production_scripts/Chapter<序号>_<篇章名>/E<NN>.md` — STEP 4 的分镜表（画面描述、声音）
   - `screenplays/Chapter<序号>_<篇章名>/E<NN>.md` — STEP 3 的剧本和片段拆分（补充细节）
   - `reference_images/characters_prompt.md` — 已确认的角色参考图
   - `reference_images/scenes_prompt.md` — 已确认的场景参考图
3. 为该篇章所有集数的每个片段生成关键帧，遵循**强制共享帧规则**：
   - **核心原则**：每集只有「独立首帧」和「共享帧」两种关键帧类型
   - **独立首帧**：仅 C01（每集第一个片段）有独立首帧
   - **共享帧**：C(N) 的末帧就是 C(N+1) 的首帧，是同一张图片，**只写一次 prompt**
   - **不存在任何其他独立首帧**，跨场景转场也通过共享帧过渡
   - 因此每集的关键帧列表 = 1 个独立首帧（C01） + N 个共享帧（每个片段的末帧）
   - 每个片段恰好需要 2 张图片输入：首帧（来自上一片段末帧或 C01 独立首帧）+ 本片段末帧
4. 每个关键帧标注应使用的参考图编号，**最多引用 4 张**
5. 按篇章批量生成，**等待用户确认**后写入文件

**推翻机制**：
- 单个参考图 prompt 不满意：修改该参考图
- 某角色/场景参考图整体不满意：推翻重写该角色/场景的全部参考图
- 单个关键帧 prompt 不满意：修改该片段
- 某集整体不满意：推翻重写该集
- 参考图确认后需要调整：回到 5A/5B 修改后重新生成关键帧

- 存放路径：`reference_images/character_images/CH0[N]_<角色中文名>.png`（如 `CH01_沈念薇.png`）

## 创作要求

### Prompt 七要素规范（所有 prompt 必须遵循）

每个 prompt（角色参考图、场景参考图、关键帧）都必须包含以下 7 类要素。这些要素不是可选的，缺一不可：

| 要素 | 可选值 | 说明 |
|------|--------|------|
| **美术风格** | 高精度3d建模风格 / 赛博朋克风格 / 水彩画风格 / 像素画风格 / 手绘风格 / 漫画风格 / 写实风格 / 抽象风格 | 根据 framework.md 的画风设定选择匹配项。与风格锁定保持一致 |
| **景别** | 近景 / 中景 / 远景 / 航拍 / 第一人称 | 角色参考图固定使用近景；场景参考图固定使用远景；关键帧根据 STEP 4 分镜表的景别列映射 |
| **画面事物** | 人物 / 动物 / 植物 / 建筑 / 环境 | 直接描述画面中存在的具体事物，不使用抽象概括 |
| **时间/光线** | 白天 / 夜晚 / 傍晚 / 正面光 / 侧逆光 | 每个场景/关键帧必须明确时间和光线条件。场景参考图选取最常出现的时间光线；关键帧按剧情时间设定 |
| **色调** | 冷色调 / 暖色调 / 单色调 / 复古色调 | 与 framework.md 的色调基调保持一致 |
| **构图** | 水平构图 / 对称构图 / 对点构图 | 角色参考图固定使用对称构图；场景参考图使用水平构图；关键帧根据画面内容选择 |
| **质量词** | 胶片颗粒 / 4k / 8k / 超清 / 高清 / 低清 | 角色参考图固定使用 `4k, 超清`；场景参考图固定使用 `4k, 超清`；关键帧默认使用 `4k, 超清`，特殊风格（如复古）可追加 `胶片颗粒` |

### 角色参考图规则

- **外观关键词**直接复用 STEP 4 角色参考表的转化结果（从 `framework.md` 提取）
- **外观关键词净化**：将 framework.md 中的文学化/叙事性描述转化为适合 AI 生图的视觉描述，去除容易被模型误解的内容：
  - 禁止：暗示疲劳/憔悴的身体状态描述（如"眼下有青黑""黑眼圈""脸色苍白无血色"），模型会将其渲染为明显的蓝色/紫色色块
  - 改为：用中性外观描述替代（如"皮肤白净""皮肤偏白"）
  - 禁止：抽象气质描述（如"气质冷清""眼神中有故事"），模型无法准确呈现
  - 改为：用具体视觉特征替代（如"眉眼清秀""五官立体"）
- **背景**统一使用灰色纯色背景（`灰色纯色背景`），让角色突出
- **穿搭**：每个角色只使用默认穿搭（框架中描述的主要穿着），不生成多套穿搭变体
- **Prompt 公式**：`[角色名]，[默认穿搭描述]，近景，正面光，[色调]，对称构图，[美术风格]，[风格锁定]，4k, 超清。[角色外观关键词]，表情平静，灰色纯色背景`
- **编号规则**：`CH` + 两位角色序号（如 CH01、CH02）
- **全局参数**：画面比例 9:16，分辨率 2K
- **禁止**：背景杂乱、动态姿态、多个角色在同一张参考图中、华丽背景

### 场景参考图规则

- **描述来源**：从 `framework.md` 的「主要场景」描述提取空间细节和陈设物件
- **无人物**：场景参考图中不出现任何角色
- **一景一图**：每个场景仅生成 1 张图片，不按光线/时间条件分组
- **视角选取**：选取该场景最具代表性的全景视角，能展现整体空间布局
- **单图 Prompt 公式**：`[场景名]场景参考图，远景，[时间/光线]，[色调]，水平构图，[美术风格]，[风格锁定]，4k, 超清。[视角描述]，[陈设物件细节]。无人物`
- **编号规则**：`SC` + 两位场景序号（如 SC01、SC02）
- **风格锁定**必须追加到每个 prompt 末尾

### 关键帧 Prompt 规则

- **强制共享帧规则**（核心，不可违反）：
  - **所有相邻片段必须共享关键帧**：C(N) 的末帧 = C(N+1) 的首帧，无论是否同场景
  - **唯一独立首帧**：C01（每集的第一个片段），其余片段首帧一律沿用前一 clip 末帧
  - **不存在跨场景独立首帧**：即使场景切换，也通过 C(N) 末帧的画面过渡到新场景（如末帧中人物走向新环境、末帧为转场画面等）

- **首帧图 Prompt**（仅 C01）：
  - 基于 STEP 4 分镜表画面描述的首句（静态起始画面），内容保持一致
  - 新增参考图引用标识
  - 继承 STEP 4 的所有画面描述规则（静态描述、禁止动态动词/比喻/抽象概念）

- **末帧图 Prompt**（每个片段都需要）：
  - 基于 STEP 4 分镜表的画面描述末帧状态 + STEP 3 片段描述，扩展为完整图片生成 prompt
  - 必须覆盖：人物姿态/表情、环境状态、光线色调、构图
  - 遵循七要素规范：`主体 + 姿态 + 环境 + 景别 + 时间/光线 + 色调 + 构图 + 美术风格 + 风格锁定 + 质量词`
  - 禁止动态描述（同首帧规则）
  - 禁止比喻和抽象概念（同首帧规则）
  - 角色外观关键词必须与角色参考表一致
  - 风格词必须与风格锁定一致

- **标题格式**：
  - 独立首帧（仅 C01）：`### C01 首帧（8s）`
  - 共享帧（末帧兼下一 clip 首帧）：`### C01 末帧 / C02 首帧`
  - 仅末帧（最后一个 clip）：`### C17 末帧`
  - **不存在跨场景独立首帧**，所有 C02+ 的片段首帧一律为前一 clip 末帧

- **参考图引用规则**：
  - 每个关键帧**必须主动引用参考图**；例外：纯白屏/纯黑屏等无画面的转场帧，允许参考图数量为 0
  - 每个关键帧**最多引用 4 张参考图**
  - **角色出现则必引**：片段中出场的重要角色，其首帧和末帧必须各自引用该角色的参考图
  - **场景出现则必引**：有明确场景的关键帧必须引用该场景的参考图
  - 引用格式：`编号（角色/场景-说明）`，多张用 `|` 分隔
  - 引用数量标注在括号中（如 `3/4` 表示引用了 4 张中的 3 张）
  - 无角色的纯环境/转场片段只引用 1 张场景参考图
  - 多角色片段可引用多个角色的参考图，但总数不超过 4 张
  - 引用角色参考图时，使用该角色的默认穿搭图片
  - 一个角色的同一关键帧最多引用 1 张参考图
  - 当引用总数超过 4 张时，优先保留：出场角色的参考图 > 场景参考图 > 次要角色的参考图

- **首末帧连贯性**（已由强制共享帧规则覆盖）：
  - C(N) 末帧 = C(N+1) 首帧，**所有相邻片段强制共享，无例外**
  - 跨场景时，末帧需自然过渡到新场景（如人物走到新环境、画面自然切换等）

- **首尾帧人物锚定**（继承 STEP 3/4 规则）：
  - 片段中的重要角色至少在首帧或末帧 Prompt 中有外观体现
  - 例外：纯环境/转场片段、背景群演/路人

## 质量检查清单

### 参考图检查（5A/5B 完成后）

1. **角色参考图背景纯净？** — 灰色纯色背景，无杂乱元素
2. **场景参考图无人物？** — 纯环境，无角色出现
3. **风格锁定一致？** — 所有参考图 prompt 末尾的风格锁定完全一致
4. **角色外观关键词一致？** — 同一角色在参考图 prompt 中的外观关键词完全一致
5. **角色参考图质量？** — 角色参考图 prompt 结构完整，包含角色名、穿搭、外观关键词、风格锁定
6. **七要素完整？** — 每个参考图 prompt 都包含：美术风格、景别、画面事物、时间/光线、色调、构图、质量词

### 关键帧检查（5C 完成后，逐片段自检）

7. **共享帧正确？** — C02 起所有片段首帧 = 前一 clip 末帧，仅 C01 有独立首帧
8. **主动引用了参考图？** — 每个关键帧都有参考图引用（纯白屏/纯黑屏等无画面转场帧除外）
9. **角色出现则必引？** — 片段中出场的重要角色在关键帧中都有对应的角色参考图引用
10. **场景出现则必引？** — 有明确场景的关键帧都引用了对应场景的参考图
11. **参考图引用正确？** — 标注的编号与实际角色/场景匹配
12. **关键帧是静态画面？** — 无动态动词（同 STEP 4 规则）
13. **末帧是完整 prompt？** — 不是简单的一句话描述，而是完整的图片生成 prompt（覆盖人物/环境/光线/构图）
14. **角色外观一致？** — 关键帧中的角色外观与角色参考表匹配
15. **风格一致？** — 风格词与风格锁定一致
16. **共享帧连贯？** — `C(N) 末帧 / C(N+1) 首帧` 的画面描述能同时作为 C(N) 的结束和 C(N+1) 的开始，包括跨场景过渡
17. **首尾帧人物锚定？** — 片段中的重要角色至少在关键帧 prompt 中有外观体现
18. **关键帧七要素完整？** — 每个关键帧 prompt 都包含：美术风格、景别、画面事物、时间/光线、色调、构图、质量词

所有路径基于**项目根目录**，`reference_images/` 与 `production_scripts/` 同级：

```
outputs/dramas/<标题>_<yyyyMMdd_HHmm>/
  reference_images/                       # STEP 5 产出
    characters_prompt.md                         # 角色参考图 prompt（项目级，全剧共用）

      CH02_苏念安.png                    # 苏念安（默认穿搭）
      ...
    scenes/                               # 场景参考图存放目录
      SC01_苏念安出租屋.png               # 每场景 1 张
```

- 后续篇章如需新增角色或场景，在对应文件和目录中追加
- 关键帧 prompt 按篇章分子目录，目录名格式为 `Chapter<篇章序号>_<篇章中文名>`（与 `screenplays/` 中的篇章子目录一致）

## 与上下游的关系

### 输入

| 来源 | 文件 | 用途 |
|------|------|------|
| STEP 1 | `framework.md` | 角色外观、场景描述、风格基调 |
| STEP 4 | `production_scripts/Chapter<序号>_<篇章名>/E<NN>.md` | 分镜表（画面描述、声音） |
| STEP 3 | `screenplays/Chapter<序号>_<篇章名>/E<NN>.md` | 剧本情绪、场景细节（补充末帧信息） |

### 输出

| 文件 | 内容 | 消费者 |
|------|------|--------|
| `reference_images/characters_prompt.md` | 角色参考图 prompt | 后续图片生成环节 |
| `reference_images/scenes_prompt.md` | 场景参考图组图 prompt | 后续图片生成环节 |
| `reference_images/Chapter<序号>_<篇章名>/E<NN>.md` | 关键帧 prompt（首帧+末帧） | 后续图片生成环节 |
| `reference_images/character_images/` | 角色参考图图片（1 张/角色：默认穿搭） | 后续图片生成环节 |
| `reference_images/scenes/` | 场景参考图图片（1 张/场景） | 后续图片生成环节 |

### 下游执行顺序

后续图片/视频生成应按以下顺序执行：

1. 生成角色参考图（`characters_prompt.md` 中的 prompt → 文生图 9:16 比例 → 每角色 1 张默认穿搭 → 存入 `character_images/`）
2. 生成场景参考图（`scenes_prompt.md` 中的单图 prompt → 文生图 9:16 比例 → 每场景 1 张图 → 存入 `scenes/`）
3. 步骤 1 和 2 可并行执行
4. 生成关键帧（`E<NN>.md` 中的 prompt → 按参考图数量选择生成方式）：
   - **参考图 ≥ 1 张**：使用图生图模式，将引用的参考图作为输入
   - **参考图 = 0 张**（纯白屏/纯黑屏等无画面转场帧）：使用文生图模式，直接从 prompt 生成，**不得传入任何参考图**
5. 关键帧图片可用于后续视频生成（图生视频模式，首帧图片作为输入）
