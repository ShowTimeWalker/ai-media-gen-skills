# 测试模式

## 概述

测试模式是一个精简的全自动流程，用于快速验证一个创意从框架到视频生成指令的完整链路是否通畅。从阶段三（剧本创作）开始不再与用户互动，AI 自主完成所有内容生成。

## 触发方式

在阶段一的创意入口处，用户输入「测试模式」或「test」即可触发。

## 固定参数

测试模式覆盖以下参数，**不受用户选择影响**：

| 参数 | 固定值 |
|------|--------|
| 单集时长 | 1 分钟 |
| 总集数 | 30 集 |
| 总时长 | 30 分钟 |
| 画面分辨率 | 480p |
| 画面比例 | 9:16 |
| 制作形式 | AI 漫剧 |

## 与正常流程的差异

| 环节 | 正常流程 | 测试模式 |
|------|---------|---------|
| 阶段一：框架 | 完整创意收集 + 5轮参数确认 + 深度对话 + 框架确认 | 简化创意收集（只需类型方向和基本设定） → 自动填充固定参数 → 直接生成框架 |
| 阶段二：故事线 | 2A大纲确认 + 2B全部篇章创作 | 自动生成大纲 + 仅创作第一篇章故事 |
| 阶段三：剧本 | 3A剧本生成+确认 → 3B片段拆分+确认，按篇章推进 | 自动生成第一篇章剧本 + 第一集前半集片段拆分（后半集占位） |
| 阶段四：生产脚本 | 4A全局参数确认 + 4B分镜表确认，按篇章推进 | 自动生成第一集前半集的分镜表 |
| 阶段五：图片提示词 | 5A角色参考图确认 + 5B场景参考图确认 + 5C关键帧确认 | 自动生成全部（≤5角色 + ≤8场景 + 第一集前半集关键帧） |
| 阶段六：视频指令 | 6A组装确认 + 6B指令确认 | 自动生成第一集前半集的视频指令 |
| 用户互动 | 每个环节都需要确认 | 阶段一简化互动后，全程自动直至完成 |

## 测试模式完整流程

### 阶段一：简化创意收集

1. 用户触发测试模式
2. AI 仅询问以下内容（合并为一轮对话）：
   - **类型方向**：想做哪种类型？（可参考热门灵感板）
   - **基本设定**：有没有一个大致的故事方向或情绪偏好？（没有则 AI 自行补全）
3. AI 基于用户输入 + 固定参数，自动生成完整 framework.md（画风根据类型自动匹配，不需要用户选）
4. **不等待用户确认框架**，直接进入阶段二

### 阶段二：第一篇章故事

1. 自动规划所有篇章大纲（30集约4-5个篇章）
2. 仅创作第一篇章的完整故事
3. **不等待用户确认**，直接进入阶段三

### 阶段三：第一集剧本 + 前半集片段拆分

1. 自动为第一篇章的第一集生成完整剧本
2. 将第一集拆分为前半集和后半集：
   - **前半集**：正常生成片段拆分（约4-6个片段，每个6-8秒）
   - **后半集**：用占位内容替代，格式如下：
     ```
     --- 后半集占位 ---
     [后半集片段占位，待正式流程补充]
     ```
3. **不等待用户确认**，直接进入阶段四

### 阶段四：前半集分镜表

1. 自动提取全局参数（Style Lock + Character Reference）
2. 为第一集前半集的每个片段生成结构化分镜表
3. 固定使用 480p / 9:16
4. **不等待用户确认**，直接进入阶段五

### 阶段五：图片提示词（文生图 + 图生图）

按顺序自动生成以下内容：

**1. 角色参考图 Prompt（≤5个角色）**
- 从 framework.md 核心人物中选取最重要的 ≤5 个角色
- 每角色生成1张参考图 prompt（默认穿搭，灰色纯色背景）

**2. 场景参考图 Prompt（≤8个场景）**
- 从 framework.md 主要场景中选取最重要的 ≤8 个场景
- 每场景生成1张参考图 prompt（无人物）

**3. 关键帧 Prompt（第一集前半集）**
- 为第一集前半集的每个片段生成首帧和末帧 prompt
- 遵循首帧复用规则（同场景 C(N) 末帧 = C(N+1) 首帧）
- 标注参考图引用

**4. 视频 Prompt（第一集前半集）**
- 基于关键帧和分镜表，为每个片段生成视频生成指令
- 包含：参考概览表、角色语音规格、分镜表（含首尾帧标注）

**不等待用户确认**，全部内容写入文件后输出完成报告。

## 完成报告

全部内容生成完毕后，输出以下报告：

```markdown
✅ 测试模式已完成！以下是本次生成的全部内容：

**项目**：[剧名]
**类型**：[类型]
**体量**：30集 × 1分钟 = 30分钟（测试模式固定）
**画风**：[自动匹配的画风]

**产出文件**（共 [X] 个）：
- `framework.md` — 核心框架
- `story.md` — 故事线（含大纲）
- `stories/EP01_EP<n>_*.md` — 第一篇章故事
- `screenplays/Chapter1_*/E01.md` — 第一集剧本（前半集片段拆分 + 后半集占位）
- `production_scripts/Chapter1_*/E01.md` — 第一集前半集分镜表
- `reference_images/characters_prompt.md` — [X]个角色参考图 prompt
- `reference_images/scenes_prompt.md` — [X]个场景参考图 prompt
- `reference_images/Chapter1_*/E01.md` — 第一集前半集关键帧 prompt
- `video_generation/Chapter1_*/E01.md` — 第一集前半集视频指令

**下一步**：
1. 使用提示词生成实际图片和视频
2. 如需继续完整流程，可从阶段二继续创作剩余篇章
3. 如需调整，可回到任意阶段修改后重新生成
```

## 冲突处理规则

测试模式的设定优先级**高于**正常流程中的任何环节：

- 固定参数（480p、1min×30集）覆盖用户在阶段一的选择
- 角色数量上限（≤5）覆盖 framework.md 中定义的全部角色
- 场景数量上限（≤8）覆盖 framework.md 中定义的全部场景
- 自动生成的框架不需要用户确认，直接进入下一步
- 当其他 reference 文件中的规则与测试模式冲突时，以测试模式为准
