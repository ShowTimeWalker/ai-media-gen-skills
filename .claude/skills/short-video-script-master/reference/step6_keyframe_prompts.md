# 阶段六：关键帧图片提示词

## 目标

在生成每个片段的首帧和末帧图片之前，先预生成角色参考图和场景参考图的 prompt，再基于参考图为每个片段的首帧和末帧编写图片 prompt，显著提升人物和场景的一致性。

产出格式是通用的，不绑定具体供应商，由 content-generation-workflow 在实际调用时适配。

## 背景：为什么需要参考图

STEP 5 的首帧图 Prompt 是纯文本驱动的 text-to-image prompt。实际生成时，即使 prompt 中包含详细的角色外观描述，AI 模型每次生成的角色外观仍会有差异。

通过预先生成角色参考图和场景参考图（使用 Seedream 的 `--sequential-image-generation` 一次性生成连贯图组），后续关键帧生成时使用 image-to-image 模式（参考图作为输入），可以显著提升一致性。

本 STEP 只输出结构化 prompt，不调用外部 API。实际生图由 content-generation-workflow 或 doubao/juzi skill 执行。

## 工作流程

### 6A：角色参考图 Prompt（按项目，首次执行）

1. 读取 `framework.md` 提取所有核心人物的「形象」字段
2. 读取当前篇章的剧本文件（`screenplays/<篇章>/E<NN>.md`），识别每个角色实际出现的情绪/表情
3. 从 `framework.md` 提取风格锁定（与 STEP 5 一致）
4. 为每个核心角色生成一组 **2x3 组图 prompt**（共 6 张连贯图）：
   - **第 1 行（情绪行，3 张）**：从剧本中该角色实际情绪选取 3 种，景别为半身
   - **第 2 行（服饰行，3 张）**：第 1 张为默认穿搭，其余为剧情中出现的其他穿搭，景别为近景
5. 向用户展示全部角色的参考图 prompt，**等待确认**

### 6B：场景参考图 Prompt（按项目，首次执行）

1. 读取 `framework.md` 提取所有「主要场景」的描述
2. 读取当前篇章的剧本文件，识别每个场景实际出现的光线/时间条件
3. 为每个场景生成 **2x2 组图 prompt**（共 4 张连贯图），覆盖该场景的主要视觉区域/角度
4. 同一场景如需不同光线条件（如日间和夜间），各生成一个独立的 2x2 组图
5. 场景参考图**不包含人物**，纯环境
6. 向用户展示全部场景的参考图 prompt，**等待确认**

6A 和 6B 一起向用户展示确认。后续篇章如需新增角色或场景，补充生成对应参考图 prompt 即可，无需重新生成全部。

### 6C：关键帧 Prompt（按篇章）

1. 用户选择要生成关键帧的篇章（或按顺序推进）
2. 读取以下文件：
   - `production_scripts/<篇章>/E<NN>.md` — STEP 5 的生产脚本（首帧图 Prompt、末帧描述）
   - `screenplays/<篇章>/E<NN>.md` — STEP 4 的剧本和片段拆分（补充细节）
   - `reference_images/characters.md` — 已确认的角色参考图
   - `reference_images/scenes.md` — 已确认的场景参考图
3. 为该篇章所有集数的每个片段生成：
   - **首帧图 Prompt**：基于 STEP 5 的首帧图 Prompt，增加参考图引用标识
   - **末帧图 Prompt**：基于 STEP 5 的末帧文本描述 + STEP 4B 片段描述，扩展为完整图片生成 prompt
4. 每个关键帧标注应使用的参考图单元格编号，**最多引用 4 张**
5. 按篇章批量生成，**等待用户确认**后写入文件

**推翻机制**：
- 单个参考图 prompt 不满意：修改该参考图
- 某角色/场景参考图整体不满意：推翻重写该角色/场景的全部参考图
- 单个关键帧 prompt 不满意：修改该片段
- 某集整体不满意：推翻重写该集
- 参考图确认后需要调整：回到 6A/6B 修改后重新生成关键帧

## 产出格式

### 角色参考图（`reference_images/characters.md`）

每个角色生成一个 2x3 组图（6 张连贯图），一行情绪、一行服饰：

```markdown
# 角色参考图

## 风格锁定

[复用 STEP 5 的风格锁定]

## 角色参考

### 沈墨白（CH01）

**生成模式**：2x3 组图（6 张）

| 编号 | 行列 | 类型 | 内容 |
|------|------|------|------|
| CH01-R1C1 | 1行1列 | 情绪-平静 | 半身，表情平静 |
| CH01-R1C2 | 1行2列 | 情绪-愤怒 | 半身，表情愤怒，眉头紧皱 |
| CH01-R1C3 | 1行3列 | 情绪-空白 | 半身，表情空白，眼神失焦 |
| CH01-R2C1 | 2行1列 | 服饰-深色衬衫 | 近景，深色衬衫袖子卷到小臂 |
| CH01-R2C2 | 2行2列 | 服饰-西装 | 近景，黑色西装 |
| CH01-R2C3 | 2行3列 | 服饰-休闲 | 近景，白色T恤 |

**生成 Prompt**：

沈墨白角色设定图，2行3列共6张。第一行三张展示不同表情：平静、愤怒眉头紧皱、空白眼神失焦，均为半身。第二行三张展示不同穿搭：深色衬衫袖子卷到小臂、黑色西装、白色T恤，均为近景。男性，28岁，身高183cm，偏瘦但肩膀宽，五官棱角分明，眉骨高、眼窝略深，薄唇，头发微长有一缕垂到额前，灰色纯色背景，[风格锁定]
```

**要点**：
- 一个组图 prompt 生成 6 张连贯图（使用 `--sequential-image-generation --max-images 6`）
- 第 1 行 3 张为情绪变体（半身），第 2 行 3 张为服饰变体（近景）
- 服饰行第 1 张（R2C1）必须是框架中描述的默认穿搭
- 每张图有独立编号 `CH0[N]-R行C列`，供关键帧引用

### 场景参考图（`reference_images/scenes.md`）

每个场景的每种光线条件生成一个 2x2 组图（4 张连贯图）：

```markdown
# 场景参考图

## 风格锁定

[复用 STEP 5 的风格锁定]

## 场景参考

### 苏念安出租屋（SC01）

#### 日间（SC01-day）

**生成模式**：2x2 组图（4 张）

| 编号 | 行列 | 内容 |
|------|------|------|
| SC01-day-R1C1 | 1行1列 | 室内正面全景 |
| SC01-day-R1C2 | 1行2列 | 窗台区域 |
| SC01-day-R2C1 | 2行1列 | 床铺区域 |
| SC01-day-R2C2 | 2行2列 | 门口视角 |

**生成 Prompt**：

苏念安出租屋场景参考图，2行2列共4张，日间自然光。第一行左：室内正面全景，靠墙铁架单人床，灰色薄被子叠整齐，白墙贴着旧日历。第一行右：窗台区域特写，窗台上一盆快死的多肉植物，窗户小采光差，阳光从窗户斜照进来。第二行左：床铺区域，铁架单人床，床头放着一个小台灯。第二行右：门口视角，从门口看向室内，能看到整体空间布局。无人物，[风格锁定]

#### 夜间（SC01-night）

**生成模式**：2x2 组图（4 张）

| 编号 | 行列 | 内容 |
|------|------|------|
| SC01-night-R1C1 | 1行1列 | 室内正面全景 |
| SC01-night-R1C2 | 1行2列 | 窗台区域 |
| SC01-night-R2C1 | 2行1列 | 床铺区域（台灯亮） |
| SC01-night-R2C2 | 2行2列 | 门口视角 |

**生成 Prompt**：

苏念安出租屋场景参考图，2行2列共4张，夜间昏暗光线。第一行左：室内正面全景，光线昏暗，只有窗外的路灯光微微透进来。第一行右：窗台区域，窗户透进微弱的灰白色路灯光，台灯关着。第二行左：床铺区域，床头小台灯开着，暖黄色光照亮一小块区域。第二行右：门口视角，从门口看向室内，整体很暗。无人物，[风格锁定]
```

**要点**：
- 同一场景不同光线条件作为独立组图（各自有独立的 2x2）
- 编号包含光线条件标识：`SC0[N]-[条件]-R行C列`
- 一个组图 prompt 生成 4 张连贯图（使用 `--sequential-image-generation --max-images 4`）

### 关键帧 Prompt（`reference_images/<篇章>/E<NN>.md`）

每个关键帧**必须引用具体的参考图单元格，最多 4 张**：

```markdown
# E01 关键帧 Prompt

> 集数：第1集 | 篇章：醒不来的噩梦 | 片段数：17

## 参考图索引

本集使用的参考图汇总：

| 类型 | 编号 | 用途 |
|------|------|------|
| 角色 | CH01-R1C1 | 沈墨白-情绪-平静 |
| 角色 | CH01-R1C2 | 沈墨白-情绪-愤怒 |
| 角色 | CH01-R2C1 | 沈墨白-服饰-深色衬衫 |
| 场景 | SC01-night-R1C1 | 苏念安出租屋-夜间-正面全景 |

## 片段关键帧

### C01 首帧（8s）

**参考图**（3/4）：CH01-R1C1（沈墨白-平静）| CH01-R2C1（沈墨白-深色衬衫）| SC01-night-R1C1（出租屋-夜间-全景）

**Prompt**：

沈墨白坐在昏暗房间的铁架单人床上，身体前倾，双手交叉放在膝盖上。深色衬衫袖子卷到小臂，左腕旧款机械表微微反光。表情平静但眼神有些空。室内光线昏暗，只有窗外微弱路灯光透进来，灰色薄被子叠整齐放在床尾。[风格锁定]

---

### C01 末帧

**参考图**（2/4）：CH01-R1C1（沈墨白-平静）| SC01-night-R2C1（出租屋-夜间-床铺区域）

**Prompt**：

沈墨白的面部特写，侧面角度。他的眼睛半睁着，瞳孔没有聚焦，像是在发呆。下颌线条松弛，左腕靠在膝盖上，旧款机械表在暗光中微微反光。背景是昏暗的房间，窗外的路灯光在墙壁上投下一道淡淡的光条。[风格锁定]

---
```

**要点**：
- 每个关键帧的**参考图**行标注引用数量（如 `3/4`）和每张参考图的编号 + 简要说明
- 引用格式：`编号（角色/场景-说明）`，多张用 `|` 分隔
- 最多引用 4 张，最少引用 1 张
- 无角色的纯环境/转场片段只引用场景参考图
- 多角色片段可引用多个角色的参考图，但总数不超过 4 张

## 创作要求

### 角色参考图规则

- **外观关键词**直接复用 STEP 5 角色参考表的转化结果（从 `framework.md` 提取）
- **背景**统一使用灰色纯色背景（`灰色纯色背景`），让角色突出
- **情绪行**（第 1 行，3 张）：
  - 从剧本中该角色实际出现的情绪选取 3 种
  - 景别统一为半身（胸部以上）
  - 表情类型示例：平静、愤怒、悲伤、焦急、空白、倔强、冷漠、温柔
- **服饰行**（第 2 行，3 张）：
  - 第 1 张（R2C1）必须是框架中描述的默认穿搭
  - 其余 1-2 张为剧情中明确出现的其他穿搭
  - 景别统一为近景（面部 + 上身穿搭）
- **组图 Prompt 公式**：`[角色名]角色设定图，2行3列共6张。第一行三张展示不同表情：[情绪1]、[情绪2]、[情绪3]，均为半身。第二行三张展示不同穿搭：[服饰1]、[服饰2]、[服饰3]，均为近景。[角色外观关键词]，灰色纯色背景，[风格锁定]`
- **编号规则**：`CH` + 两位角色序号 + `-R` + 行号 + `C` + 列号（如 CH01-R1C2）
- **禁止**：背景杂乱、动态姿态、多个角色在同一张参考图中、华丽背景

### 场景参考图规则

- **描述来源**：从 `framework.md` 的「主要场景」描述提取空间细节和陈设物件
- **无人物**：场景参考图中不出现任何角色
- **光线区分**：同一场景在不同光线/时间条件下生成独立的 2x2 组图
  - 常见条件：日间、夜间、黄昏、清晨、阴天
  - 从剧本中该场景实际出现的光线条件确定需要哪些变体
  - 不同光线变体各自有独立编号前缀（如 `SC01-day`、`SC01-night`）
- **角度选取**：4 张图覆盖该场景的主要视觉区域
  - 典型组合：正面全景 + 特定区域特写 + 另一区域 + 入口/远景视角
  - 选取标准：覆盖剧本中该场景实际需要用到的视角
- **组图 Prompt 公式**：`[场景名]场景参考图，2行2列共4张，[光线条件]。第一行左：[角度1描述]。第一行右：[角度2描述]。第二行左：[角度3描述]。第二行右：[角度4描述]。无人物，[风格锁定]`
- **编号规则**：`SC` + 两位场景序号 + `-` + 光线条件 + `-R` + 行号 + `C` + 列号（如 SC01-day-R1C2）
- **风格锁定**必须追加到每个 prompt 末尾

### 关键帧 Prompt 规则

- **首帧图 Prompt**：
  - 基于 STEP 5 的首帧图 Prompt，内容保持一致
  - 新增参考图引用标识
  - 继承 STEP 5 的所有首帧图规则（静态描述、Seedream 规范、禁止动态动词/比喻/抽象概念）

- **末帧图 Prompt**（核心新增）：
  - 基于 STEP 5 的末帧文本描述 + STEP 4B 片段描述，扩展为完整图片生成 prompt
  - 必须覆盖：人物姿态/表情、环境状态、光线色调、构图
  - 遵循 Seedream 规范：`主体 + 姿态 + 环境 + 风格/光影/构图`
  - 禁止动态描述（同首帧规则）
  - 禁止比喻和抽象概念（同首帧规则）
  - 角色外观关键词必须与角色参考表一致
  - 风格词必须与风格锁定一致

- **参考图引用规则**：
  - 每个关键帧**必须主动引用参考图**，不允许无引用的关键帧
  - 每个关键帧**最多引用 4 张参考图**
  - **角色出现则必引**：片段中出场的重要角色，其首帧和末帧必须各自引用该角色的参考图（优先 1 情绪 + 1 服饰）
  - **场景出现则必引**：有明确场景的关键帧必须引用该场景的参考图
  - 引用格式：`编号（角色/场景-说明）`，多张用 `|` 分隔
  - 引用数量标注在括号中（如 `3/4` 表示引用了 4 张中的 3 张）
  - 无角色的纯环境/转场片段只引用 1 张场景参考图
  - 多角色片段可引用多个角色的参考图，但总数不超过 4 张
  - 引用角色参考图时，优先选择与当前片段表情最匹配的情绪行（第 1 行）
  - 引用角色参考图时，优先选择与当前片段穿搭匹配的服饰行（第 2 行）
  - 一个角色的同一关键帧最多引用 2 张（1 情绪 + 1 服饰）
  - 当引用总数超过 4 张时，优先保留：出场角色的参考图 > 场景参考图 > 次要角色的参考图

- **连贯性**：
  - C(N) 的末帧 Prompt 与 C(N+1) 的首帧 Prompt 应能自然衔接
  - 同场景内相邻片段：末帧和首帧的画面描述应接近（同机位或自然过渡）
  - 跨场景转场：首帧为新场景的起始构图

- **首尾帧人物锚定**（继承 STEP 4B/5 规则）：
  - 片段中的重要角色至少在首帧或末帧 Prompt 中有外观体现
  - 例外：纯环境/转场片段、背景群演/路人

## 质量检查清单

### 参考图检查（6A/6B 完成后）

1. **角色参考图背景纯净？** — 灰色纯色背景，无杂乱元素
2. **场景参考图无人物？** — 纯环境，无角色出现
3. **情绪覆盖完整？** — 剧本中该角色实际出现的核心情绪在情绪行中都有对应
4. **光线条件覆盖完整？** — 剧本中该场景实际出现的光线变体都有独立组图
5. **风格锁定一致？** — 所有参考图 prompt 末尾的风格锁定完全一致
6. **角色外观关键词一致？** — 同一角色在组图 prompt 中的外观关键词完全一致
7. **组图连贯性？** — 角色组图内 6 张外观统一、场景组图内 4 张风格统一

### 关键帧检查（6C 完成后，逐片段自检）

8. **主动引用了参考图？** — 每个关键帧都有参考图引用，无引用的关键帧不允许通过
9. **角色出现则必引？** — 片段中出场的重要角色在首帧和末帧中都有对应的角色参考图引用
10. **场景出现则必引？** — 有明确场景的关键帧都引用了对应场景的参考图
11. **参考图引用正确？** — 标注的编号与实际角色/场景匹配
12. **首帧是静态画面？** — 无动态动词（同 STEP 5 规则）
13. **末帧是完整 prompt？** — 不是简单的一句话描述，而是完整的图片生成 prompt（覆盖人物/环境/光线/构图）
14. **角色外观一致？** — 关键帧中的角色外观与角色参考表匹配
15. **风格一致？** — 风格词与风格锁定一致
16. **连贯性？** — 末帧与下一个片段的首帧能自然衔接
17. **首尾帧人物锚定？** — 片段中的重要角色至少在首帧或末帧 prompt 中有外观体现

## 文件输出

### 目录结构

所有路径基于**项目根目录**，`reference_images/` 与 `production_scripts/` 同级：

```
outputs/scripts/<标题>_<yyyyMMdd>/
  reference_images/                       # STEP 6 产出
    characters.md                         # 角色参考图 prompt（项目级，全剧共用）
```

## 与上下游的关系

### 输入

| 来源 | 文件 | 用途 |
|------|------|------|
| STEP 2 | `framework.md` | 角色外观、场景描述、风格基调 |
| STEP 5 | `production_scripts/<篇章>/E<NN>.md` | 首帧图 Prompt、末帧描述 |
| STEP 4 | `screenplays/<篇章>/E<NN>.md` | 剧本情绪、场景细节（补充末帧信息） |

### 输出

| 文件 | 内容 | 消费者 |
|------|------|--------|
| `reference_images/characters.md` | 角色参考图组图 prompt | content-generation-workflow（先生成参考图） |
| `reference_images/scenes.md` | 场景参考图组图 prompt | content-generation-workflow（先生成参考图） |
| `reference_images/<篇章>/E<NN>.md` | 关键帧 prompt（首帧+末帧） | content-generation-workflow（用参考图生成关键帧） |
| `reference_images/characters/CH0[N]_*/` | 角色参考图图片（6 张/角色） | content-generation-workflow（作为关键帧的 image-to-image 输入） |
| `reference_images/scenes/SC0[N]_*/` | 场景参考图图片（4 张/光线条件） | content-generation-workflow（作为关键帧的 image-to-image 输入） |

### 下游执行顺序

content-generation-workflow 应按以下顺序执行：

1. 生成角色参考图（`characters.md` 中的组图 prompt → Seedream `--sequential-image-generation --max-images 6` → 6 张连贯图 → 存入 `characters/CH0[N]_角色名/`）
2. 生成场景参考图（`scenes.md` 中的组图 prompt → Seedream `--sequential-image-generation --max-images 4` → 4 张连贯图 → 存入 `scenes/SC0[N]_场景名/[光线条件]/`）
3. 步骤 1 和 2 可并行执行
4. 生成关键帧（`E<NN>.md` 中的 prompt → Seedream image-to-image，将引用的参考图作为 `--image` 输入，最多 4 张 → 得到关键帧图片）
5. 关键帧图片可用于后续视频生成（Seedance I2V，`--image-url --role first_frame`）
