# 阶段六：关键帧图片提示词

## 目标

在生成每个片段的首帧和末帧图片之前，先预生成角色参考图和场景参考图的 prompt，再基于参考图为每个片段的首帧和末帧编写图片 prompt，显著提升人物和场景的一致性。

产出格式是通用的，不绑定具体供应商，由 content-generation-workflow 在实际调用时适配。

## 背景：为什么需要参考图

STEP 5 的首帧图 Prompt 是纯文本驱动的 text-to-image prompt。实际生成时，即使 prompt 中包含详细的角色外观描述，AI 模型每次生成的角色外观仍会有差异。

通过预先生成角色服饰参考图和场景参考图（使用 Seedream 的 `--sequential-image-generation` 一次性生成连贯图组），后续关键帧生成时使用 image-to-image 模式（参考图作为输入），可以显著提升一致性。

本 STEP 只输出结构化 prompt，不调用外部 API。实际生图由 content-generation-workflow 或 doubao/juzi skill 执行。

## 工作流程

### 前置校验（每次进入 STEP 6 时执行）

在开始生成之前，**必须**验证以下前置条件是否满足：

| 校验项 | 依赖来源 | 校验内容 | 不通过时处理 |
|--------|---------|---------|-------------|
| framework.md 存在 | STEP 2 | 文件存在且包含「核心人物」形象字段和「主要场景」描述 | 提示用户回到 STEP 2 生成核心框架 |
| 生产脚本存在 | STEP 5 | `production_scripts/<篇章>/E<NN>.md` 对应该篇章所有集数都存在 | 提示用户回到 STEP 5 生成生产脚本 |
| 生产脚本内容完整 | STEP 5 | 每集文件中包含「首帧图 Prompt」和「末帧」字段 | 提示用户回到 STEP 5 补充完整生产脚本 |
| 剧本文件存在 | STEP 4 | `screenplays/<篇章>/E<NN>.md` 对应该篇章所有集数都存在 | 提示用户回到 STEP 4 生成剧本 |

校验流程：
1. 用 Read 工具读取 `framework.md`，确认包含「核心人物」中每个角色的「形象」字段和「主要场景」表格
2. 用 Glob 工具扫描 `production_scripts/<篇章>/` 目录，确认集数完整
3. 用 Grep 工具检查每集生产脚本中是否包含 `首帧图 Prompt` 和 `末帧` 字段
4. 用 Glob 工具扫描 `screenplays/<篇章>/` 目录作为辅助参考
5. 如果任何校验项不通过，向用户明确告知缺失内容和建议的回退步骤，**不要继续生成**

### 6A：角色参考图 Prompt（按项目，首次执行）

1. 读取 `framework.md` 提取所有核心人物的「形象」字段
2. 读取当前篇章的剧本文件（`screenplays/<篇章>/E<NN>.md`），识别每个角色实际出现的穿搭
3. 从 `framework.md` 提取风格锁定（与 STEP 5 一致）
4. 为每个核心角色生成 1 张组图 prompt：
   - **服饰组图**（2x2，4 种穿搭）：第 1 张为默认穿搭，其余为剧情中出现的其他穿搭，景别为近景
5. 向用户展示全部角色的参考图 prompt，**等待确认**

### 6B：场景参考图 Prompt（按项目，首次执行）

1. 读取 `framework.md` 提取所有「主要场景」的描述
2. 为每个场景生成 **1 张单图 prompt**，选取该场景最具代表性的视角
3. 场景参考图**不包含人物**，纯环境
4. 向用户展示全部场景的参考图 prompt，**等待确认**

6A 和 6B 一起向用户展示确认。后续篇章如需新增角色或场景，补充生成对应参考图 prompt 即可，无需重新生成全部。

### 6C：关键帧 Prompt（按篇章）

1. 用户选择要生成关键帧的篇章（或按顺序推进）
2. 读取以下文件：
   - `production_scripts/<篇章>/E<NN>.md` — STEP 5 的生产脚本（首帧图 Prompt、末帧描述）
   - `screenplays/<篇章>/E<NN>.md` — STEP 4 的剧本和片段拆分（补充细节）
   - `reference_images/characters_prompt.md` — 已确认的角色参考图
   - `reference_images/scenes_prompt.md` — 已确认的场景参考图
3. 为该篇章所有集数的每个片段生成：
   - **首帧图 Prompt**：基于 STEP 5 的首帧图 Prompt，增加参考图引用标识
   - **末帧图 Prompt**：基于 STEP 5 的末帧文本描述 + STEP 4B 片段描述，扩展为完整图片生成 prompt
4. 每个关键帧标注应使用的参考图单元格编号，**最多引用 4 张**
5. 按篇章批量生成，**等待用户确认**后写入文件

**推翻机制**：
- 单个参考图 prompt 不满意：修改该参考图
- 某角色/场景参考图整体不满意：推翻重写该角色/场景的全部参考图
- 单个关键帧 prompt 不满意：修改该片段
- 某集整体不满意：推翻重写该集
- 参考图确认后需要调整：回到 6A/6B 修改后重新生成关键帧

## 创作要求

### 角色参考图规则

- **外观关键词**直接复用 STEP 5 角色参考表的转化结果（从 `framework.md` 提取）
- **外观关键词净化**：将 framework.md 中的文学化/叙事性描述转化为适合 AI 生图的视觉描述，去除容易被模型误解的内容：
  - 禁止：暗示疲劳/憔悴的身体状态描述（如"眼下有青黑""黑眼圈""脸色苍白无血色"），模型会将其渲染为明显的蓝色/紫色色块
  - 改为：用中性外观描述替代（如"皮肤白净""皮肤偏白"）
  - 禁止：抽象气质描述（如"气质冷清""眼神中有故事"），模型无法准确呈现
  - 改为：用具体视觉特征替代（如"眉眼清秀""五官立体"）
- **背景**统一使用灰色纯色背景（`灰色纯色背景`），让角色突出
- **服饰组图**（2x2，4 种穿搭）：
  - 第 1 张（R1C1）必须是框架中描述的默认穿搭
  - 其余为剧情中明确出现的其他穿搭
  - 景别统一为近景（面部 + 上身穿搭），表情统一为平静
  - 编号前缀：`-O-`（Outfit）
- **组图 Prompt 公式（服饰）**：`[角色名]角色服饰设定图，2行2列共4张，均为近景，表情平静。第一行左：[服饰1]。第一行右：[服饰2]。第二行左：[服饰3]。第二行右：[服饰4]。[角色外观关键词]，灰色纯色背景，[风格锁定]`
- **编号规则**：`CH` + 两位角色序号 + `-O-` + `R` + 行号 + `C` + 列号（如 CH01-O-R2C1）
- **全局参数**：画面比例 9:16，分辨率 2K
- **禁止**：背景杂乱、动态姿态、多个角色在同一张参考图中、华丽背景

### 场景参考图规则

- **描述来源**：从 `framework.md` 的「主要场景」描述提取空间细节和陈设物件
- **无人物**：场景参考图中不出现任何角色
- **一景一图**：每个场景仅生成 1 张图片，不按光线/时间条件分组
- **视角选取**：选取该场景最具代表性的全景视角，能展现整体空间布局
- **单图 Prompt 公式**：`[场景名]场景参考图，[视角描述]，[陈设物件细节]。无人物，[风格锁定]`
- **编号规则**：`SC` + 两位场景序号（如 SC01、SC02）
- **风格锁定**必须追加到每个 prompt 末尾

### 关键帧 Prompt 规则

- **首帧图 Prompt**：
  - 基于 STEP 5 的首帧图 Prompt，内容保持一致
  - 新增参考图引用标识
  - 继承 STEP 5 的所有首帧图规则（静态描述、Seedream 规范、禁止动态动词/比喻/抽象概念）

- **末帧图 Prompt**（核心新增）：
  - 基于 STEP 5 的末帧文本描述 + STEP 4B 片段描述，扩展为完整图片生成 prompt
  - 必须覆盖：人物姿态/表情、环境状态、光线色调、构图
  - 遵循 Seedream 规范：`主体 + 姿态 + 环境 + 风格/光影/构图`
  - 禁止动态描述（同首帧规则）
  - 禁止比喻和抽象概念（同首帧规则）
  - 角色外观关键词必须与角色参考表一致
  - 风格词必须与风格锁定一致

- **参考图引用规则**：
  - 每个关键帧**必须主动引用参考图**，不允许无引用的关键帧
  - 每个关键帧**最多引用 4 张参考图**
  - **角色出现则必引**：片段中出场的重要角色，其首帧和末帧必须各自引用该角色的服饰参考图
  - **场景出现则必引**：有明确场景的关键帧必须引用该场景的参考图
  - 引用格式：`编号（角色/场景-说明）`，多张用 `|` 分隔
  - 引用数量标注在括号中（如 `3/4` 表示引用了 4 张中的 3 张）
  - 无角色的纯环境/转场片段只引用 1 张场景参考图
  - 多角色片段可引用多个角色的参考图，但总数不超过 4 张
  - 引用角色参考图时，优先选择与当前片段穿搭匹配的服饰组图子图（`-O-` 前缀）
  - 一个角色的同一关键帧最多引用 1 张服饰参考图
  - 当引用总数超过 4 张时，优先保留：出场角色的参考图 > 场景参考图 > 次要角色的参考图

- **首末帧连贯性**（硬性要求）：
  - C(N+1) 的首帧必须是 C(N) 末帧的延续——角色状态（表情、姿态、位置）、场景细节、光影完全一致
  - 编写 C(N+1) 首帧 Prompt 时，**必须先读取 C(N) 末帧 Prompt**，确保画面无缝衔接
  - 允许的例外：
    - **跨场景转场**：C(N) 和 C(N+1) 为不同场景，首帧为新场景的起始画面（如闪白转场、地点切换）
    - **运镜切换**：剧本明确标注了机位/景别变化的片段（如全景切特写），允许构图变化，但角色状态（表情、姿态）必须一致
  - 禁止：同场景相邻片段出现角色表情前后矛盾、位置跳变、道具状态不一致

- **首尾帧人物锚定**（继承 STEP 4B/5 规则）：
  - 片段中的重要角色至少在首帧或末帧 Prompt 中有外观体现
  - 例外：纯环境/转场片段、背景群演/路人

## 质量检查清单

### 参考图检查（6A/6B 完成后）

1. **角色参考图背景纯净？** — 灰色纯色背景，无杂乱元素
2. **场景参考图无人物？** — 纯环境，无角色出现
3. **风格锁定一致？** — 所有参考图 prompt 末尾的风格锁定完全一致
4. **角色外观关键词一致？** — 同一角色在组图 prompt 中的外观关键词完全一致
5. **组图连贯性？** — 角色组图内 4 张外观统一

### 关键帧检查（6C 完成后，逐片段自检）

7. **主动引用了参考图？** — 每个关键帧都有参考图引用，无引用的关键帧不允许通过
8. **角色出现则必引？** — 片段中出场的重要角色在首帧和末帧中都有对应的角色参考图引用
9. **场景出现则必引？** — 有明确场景的关键帧都引用了对应场景的参考图
10. **参考图引用正确？** — 标注的编号与实际角色/场景匹配
11. **首帧是静态画面？** — 无动态动词（同 STEP 5 规则）
12. **末帧是完整 prompt？** — 不是简单的一句话描述，而是完整的图片生成 prompt（覆盖人物/环境/光线/构图）
13. **角色外观一致？** — 关键帧中的角色外观与角色参考表匹配
14. **风格一致？** — 风格词与风格锁定一致
15. **首末帧连贯？** — C(N+1) 首帧与 C(N) 末帧的角色状态（表情、姿态、位置）、场景细节完全一致（跨场景转场和明确运镜切换除外）
16. **首尾帧人物锚定？** — 片段中的重要角色至少在首帧或末帧 prompt 中有外观体现

```
      ...
```

- `characters_prompt.md` 和 `scenes_prompt.md` 是项目级 prompt 文件，全剧共用
- `character_images/` 是角色参考图存放目录，所有角色的图片平铺在该目录下
- `scenes/` 是场景参考图存放目录，所有场景图片平铺在该目录下，每个场景 1 张
- 后续篇章如需新增角色或场景，在对应文件和目录中追加
- 关键帧 prompt 按篇章分子目录，目录名与 `screenplays/` 中的篇章子目录一致

告知用户："篇章「[篇章名]」的关键帧 prompt 已保存到 `outputs/scripts/<标题>_<yyyyMMdd>/reference_images/<篇章缩写>/`。是否继续下一个篇章？"

## 与上下游的关系

### 输入

| 来源 | 文件 | 用途 |
|------|------|------|
| STEP 2 | `framework.md` | 角色外观、场景描述、风格基调 |
| STEP 5 | `production_scripts/<篇章>/E<NN>.md` | 首帧图 Prompt、末帧描述 |
| STEP 4 | `screenplays/<篇章>/E<NN>.md` | 剧本情绪、场景细节（补充末帧信息） |

### 输出

| 文件 | 内容 | 消费者 |
|------|------|--------|
| `reference_images/characters_prompt.md` | 角色参考图组图 prompt | content-generation-workflow（先生成参考图） |
| `reference_images/scenes_prompt.md` | 场景参考图组图 prompt | content-generation-workflow（先生成参考图） |
| `reference_images/<篇章>/E<NN>.md` | 关键帧 prompt（首帧+末帧） | content-generation-workflow（用参考图生成关键帧） |
| `reference_images/character_images/` | 角色参考图图片（1 张/角色：服饰） | content-generation-workflow（作为关键帧的 image-to-image 输入） |
| `reference_images/scenes/` | 场景参考图图片（1 张/场景） | content-generation-workflow（作为关键帧的 image-to-image 输入） |

### 下游执行顺序

content-generation-workflow 应按以下顺序执行：

1. 生成角色参考图（`characters_prompt.md` 中的组图 prompt → Seedream `--sequential-image-generation --max-images 4 --ratio 9:16`，2K 分辨率 → 每角色 1 张合成图 → 存入 `character_images/`）
2. 生成场景参考图（`scenes_prompt.md` 中的单图 prompt → Seedream `--ratio 9:16`，2K 分辨率 → 每场景 1 张图 → 存入 `scenes/`）
3. 步骤 1 和 2 可并行执行
4. 生成关键帧（`E<NN>.md` 中的 prompt → Seedream image-to-image，将引用的参考图作为 `--image` 输入，最多 4 张 → 得到关键帧图片）
5. 关键帧图片可用于后续视频生成（Seedance I2V，`--image-url --role first_frame`）
