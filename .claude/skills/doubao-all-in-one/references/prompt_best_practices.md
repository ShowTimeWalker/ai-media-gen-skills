# 提示词最佳实践规范

本文件包含图片、视频（Seedance）、即梦视频三类生成的提示词审核规则。

**默认行为**：调用生成脚本前，自动按以下规则审核并优化用户提示词。用户明确指定"不优化""直接生成""跳过优化"时跳过。

---

## 通用规则

- **文件名标题**：优化提示词时同时生成一个简短标题，不超过 10 个中文字，通过 `--name` 参数传入
- **自动优化格式**：
  ```
  优化说明：
  - [具体补充了什么]

  文件名标题：[不超过 10 个中文字的内容简述]

  优化后提示词：
  [完整提示词]
  ```

## 图片生成（Seedream）

**参考文档**：`references/seedream_prompt_guide.md`

### 提示词公式

主体 + 行为（非必须）+ 环境（非必须）+ 风格/色彩/光影/构图（非必须）

### 基本原则

- 使用自然语言描述，不要用逗号分隔的关键词堆叠
- 主体必须明确，特征具体

### 参数约束

无特殊约束，按用户指定或脚本默认值执行。

### 质量检查清单

- [ ] **使用自然语言**：是否用简洁连贯的句子描述画面，而非逗号分隔的关键词堆叠？
- [ ] **主体明确**：是否清楚描述了画面中的主体是谁/什么？特征是否具体？
- [ ] **编辑目标明确**（图生图）：是否准确指示了需要编辑的对象和变化要求？是否避免使用模糊代词？
- [ ] **保持不变的部分**（图生图）：是否明确说明了希望保持不变的内容？
- [ ] **文字内容加引号**：如需生成文字，文字内容是否放在双引号中？
- [ ] **简洁精确**（5.0 lite/4.5/4.0）：是否避免了不必要的华丽词汇堆叠？简洁精确优于重复冗余
- [ ] **风格词精准**：如有风格需求，是否使用了精准的风格词而非模糊描述？
- [ ] **多图关系清晰**（多图输入）：是否清楚指明了不同图像间的编辑/参考关系？

### 必须拒绝的情况

- 主体完全缺失（如"一张好看的图""生成一张图"）

### 应自动补充的情况

- 关键词堆砌 → 改写为自然语言
- 缺少风格/美学描述 → 根据画面内容补充合适的风格
- 图生图中使用模糊代词 → 替换为具体对象描述
- 需要渲染文字但未加双引号 → 补充双引号
- 多图输入但未指明图间关系 → 根据意图补充关系描述

### 图生图额外规则

- **图像编辑**（增加/删除/替换/修改）：用简洁明确的指令，准确指示编辑对象
- **参考图生图**：提示词必须包含两部分——**指明参考对象** + **描述生成画面**
- **多图输入**：清楚指明不同图像需要编辑/参考的对象及操作（如"用图一的人物替换图二的人物"）
- **设计草图转高保真**：注明"遵循图中文字内容进行生成"，明确需与参考图保持一致的关键要求

## 视频生成（Seedance 1.5 Pro）

**参考文档**：`references/seedance_1_5_pro_prompt_guide.md`

### 提示词公式

主体 + 运动 + 环境（非必须）+ 运镜/切镜（非必须）+ 美学描述（非必须）+ 声音（非必须）

### 参数约束

- **分辨率默认 480p**：除非用户明确要求更高分辨率（如"720p""1080p""高清"），否则不得传 `--resolution` 参数
- **宽高比默认 16:9**：除非用户明确要求其他比例（如"竖屏""正方形""9:16""1:1"等），否则不得传 `--ratio` 参数。有首帧/尾帧参考图且用户未指定比例时也应使用 16:9，而非 adaptive

### 质量检查清单

- [ ] **主体明确**：是否清楚描述了主体是谁/什么？外貌、穿着、姿态是否具体？
- [ ] **运动描述**：是否描述了主体的动作和运动方式？是否使用了程度副词控制节奏？
- [ ] **角色特征一致**：多人场景中，每个角色的特征描述是否全文一致、可唯一区分？
- [ ] **切镜清晰**（如有切镜）：是否明确标注了镜头编号？景别/内容是否有区分？切镜时机是否精准？
- [ ] **对话/声音**（如需音频）：是否描述了音色特征？是否指定了语言类型？对话内容是否具体？
- [ ] **运镜描述**（如有运镜）：是否符合"起幅 + 运镜 + 幅度 + 落幅"公式？是否使用了专业术语？
- [ ] **环境描述**（如需）：是否描述了光线、天气、氛围等环境信息？
- [ ] **风格参考**（如需）：是否指定了美学风格或参考作品？

### 必须拒绝的情况

- 主体完全缺失（如"生成一个视频""做个动画"）
- 主体不明确且无法推断（如"一个东西在动"）

### 应自动补充的情况

- 缺少运动描述 → 根据主体特性补充合理的动作和运动方式
- 缺少环境描述 → 根据主体和场景补充环境信息
- 多人场景角色特征不可区分 → 为每个角色补充唯一标识特征
- 包含切镜但未区分镜头编号或景别 → 补充镜头编号和景别描述
- 包含对话但未指定语言类型或音色特征 → 补充音色和语言描述

## 视频生成（即梦 3.0）

**参考文档**：`references/jimeng_3_0_prompt_guide.md`

### 提示词公式

主体 + 运动 + 环境（非必须）+ 运镜/切镜（非必须）+ 美学描述（非必须）

> 即梦 3.0 不支持声音生成，提示词应专注于视觉画面。建议中英文均可，不超过 400 字。

### 参数约束

- **分辨率默认 720p**：除非用户明确要求 1080p（如"高清""1080p""全高清"），否则不得传 `--resolution` 参数
- **不支持手动指定宽高比**：即梦 3.0 文生视频有 `--aspect-ratio` 参数（默认 16:9），图生视频由图片决定宽高比

### 质量检查清单

- [ ] **主体明确**：是否清楚描述了主体是谁/什么？外貌、穿着、姿态是否具体？
- [ ] **运动描述**：是否描述了主体的动作和运动方式？是否使用了程度副词控制节奏？
- [ ] **角色特征一致**：多人场景中，每个角色的特征描述是否全文一致、可唯一区分？
- [ ] **切镜清晰**（如有切镜）：是否明确标注了镜头编号？景别/内容是否有区分？切镜时机是否精准？
- [ ] **运镜描述**（如有运镜）：是否符合"起幅 + 运镜 + 幅度 + 落幅"公式？是否使用了专业术语？
- [ ] **环境描述**（如需）：是否描述了光线、天气、氛围等环境信息？
- [ ] **风格参考**（如需）：是否指定了美学风格或参考作品？
- [ ] **无声音描述**：即梦 3.0 不支持音频，是否避免了对话/音效/BGM 相关描述？
- [ ] **图生视频一致性**（如使用参考图）：提示词是否与首帧/尾帧图片内容保持一致？

### 必须拒绝的情况

- 包含切镜但未区分镜头编号或景别 → 补充镜头编号和景别描述
- 包含声音/对话/音效描述 → 移除声音相关描述，仅保留视觉内容
