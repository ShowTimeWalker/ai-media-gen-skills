# 番茄向写作与运营习惯（基线）

> 以下为可执行的通用实践归纳，便于自检与改稿。具体规则与算法以平台最新公示为准。

## 1. 书名、简介与封面

- **书名**：尽量在类型清晰的前提下有记忆点；避免空泛堆砌热词；与正文气质一致，避免点进来发现「货不对板」。
- **简介**：前两三句给出**人设 / 处境 / 核心矛盾 / 独特设定**；少用抽象形容词堆叠；可留一句悬念，但不要全文剧透关键谜底。
- **封面**：信息完整、与书名作者一致；避免侵权素材与违规元素（细则见 `compliance_and_format.md`）。

## 2. 开篇与首秀阶段

- **前若干章**：尽快让读者理解「这是什么故事、主角是谁、麻烦是什么、凭什么要看下去」。
- **冲突与信息**：每章建议有可见的推进或情绪起伏，避免长段说明书式设定倾泻。
- **章末**：常用做法是留一个小钩子（信息、危机、反转、抉择），但避免机械重复同一种结尾腔。

## 3. 稳定更新与连载节奏

- 稳定日更或固定频率有助于读者形成习惯；断更过久会伤追读。
- 单章体量以读者一口气读完舒适为宜；过长注意分段与场景切换，避免视觉疲劳。

## 4. 标签与分类

- 标签与分类应服务「找书预期」：读者点进来能获得预期类型的体验。
- 避免为了流量打与正文不符的标签。

## 5. 互动与评论（如有）

- 对反馈分类处理：合理建议可吸收；情绪化言论不纠缠；涉及审核与侵权问题按平台流程处理。

## 6. 数据与心态

- 读完率、书架、催更等是结果指标，不是唯一目标；优先把「故事是否清楚、人物是否立住、节奏是否舒服」做好，再对照数据做**小步实验**（换开篇、换简介、调整章末钩子），避免频繁大改导致风格漂移。
