# 阶段一：创意入口与核心框架

## 目标

把用户的模糊想法收敛为可执行的小说核心框架，并输出 `framework.md` 作为全流程锚定文件。

## 输入

- 用户初始创意（可为空）
- 用户偏好（题材、读者、风格等）

## 输出

- `framework.md`

## 必须确认的 6 个维度

1. 题材方向（如都市、仙侠、悬疑、言情、科幻）
2. 目标读者（女频/男频/泛读者）
3. 叙事视角（第一人称/第三人称/多视角）
4. 文风（克制、热血、轻松、冷峻、诗性等）
5. 核心情绪（爽感、治愈、压迫、燃、甜虐）
6. 一句话灵感（故事最小可行表达）

若用户信息不足，先给 2-3 个方向供其选择，再继续追问。

## 推荐开场话术

```markdown
你好，我是小说工作流大师。

先给我一句话灵感就行：一个画面、一个人物困境，或者一个你想写的情绪。

如果你还没想好，我也可以先给你 3 个方向让你选。

需要快速验证流程可输入「测试模式」。
需要保持完整构思但后续自动跑完，可输入「全自动模式」。
```

## 门禁规则

- 未确认 6 个维度前，不进入阶段二。
- 阶段一完成后必须请求用户确认：

```markdown
阶段一已完成，`framework.md` 已生成。请选择下一步：
1. 正常进入阶段二（规格与结构骨架）
2. 切换全自动推进（阶段二至阶段七自动完成）
3. 修改框架后再继续
```

## 全自动化模式触发（阶段一）

触发方式：

- 开场关键词触发：用户输入「全自动模式」「自动跑完」「full-auto」
- 阶段一结束时选择“切换全自动推进”

触发后规则：

- 阶段一仍按正常流程完成，不跳过构思与确认
- 阶段二起自动推进到阶段七，不逐步等待用户确认
- 篇幅不固定，沿用用户已确认规格
