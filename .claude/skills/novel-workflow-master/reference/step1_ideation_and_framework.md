# 阶段一：创意入口与核心框架

## 目标

把用户的模糊想法收敛为可执行的小说核心框架，并输出 `framework.md` 作为全流程锚定文件。

## 输入

- 用户初始创意（可为空）
- 用户偏好（题材、读者、风格等）

## 输出

- `framework.md`

## 必须确认的 6 个维度

1. 题材方向（如都市、仙侠、悬疑、言情、科幻）
2. 目标读者（女频/男频/泛读者）
3. 叙事视角（第一人称/第三人称/多视角）
4. 文风（克制、热血、轻松、冷峻、诗性等）
5. 核心情绪（爽感、治愈、压迫、燃、甜虐）
6. 一句话灵感（故事最小可行表达）

若用户信息不足，先给 2-3 个方向供其选择，再继续追问。

## 推荐开场话术

```markdown
你好，我是小说工作流大师。

为了尽量减少输入，你可以直接回数字：
0. 我没想法，先给我热门灵感参考（默认推荐）
1. 我有一句话灵感，进入正常构思
2. 测试模式（固定规模自动跑完）
3. 全自动模式（阶段一正常，后续自动推进）
9. 自定义输入
```

## 门禁规则

- 未确认 6 个维度前，不进入阶段二。
- 阶段一完成后必须请求用户确认：

```markdown
阶段一已完成，`framework.md` 已生成。请选择下一步：
0. 使用默认推荐（正常进入阶段二）
1. 正常进入阶段二（规格与结构骨架）
2. 切换全自动推进（阶段二至阶段七自动完成）
3. 修改框架后再继续
9. 自定义说明
```

## 全自动化模式触发（阶段一）

触发方式：

- 开场关键词触发：用户输入「全自动模式」「自动跑完」「full-auto」
- 阶段一结束时选择编号 `2`（切换全自动推进）

触发后规则：

- 阶段一仍按正常流程完成，不跳过构思与确认
- 阶段二起自动推进到阶段七，不逐步等待用户确认
- 篇幅不固定，沿用用户已确认规格

## 番茄热门灵感参考（阶段一可选）

当用户出现以下意图时，应先查阅 `reference/tomato_hot_themes.md` 再继续提问：

- “没想法/随便来一个/给我灵感”
- “按番茄热门来”
- “男频现在什么火/女频现在什么火”

推荐输出方式：

1. 先给“男频 TOP 方向 + 女频 TOP 方向”各 3 个
2. 再给“可直接开写的一句话钩子”各 1 条
3. 最后让用户选方向，进入 6 维确认
4. 若用户输入 `0`，自动采用默认推荐方向并继续

示例引导话术：

```markdown
我先给你一版“番茄近期热门灵感参考”，你选一个最有感觉的方向：

男频方向（3选1）：
1) 都市脑洞+规则博弈
2) 末世/游戏入侵+生存升级
3) 修仙2.0（现代语境/组织化修仙）

女频方向（3选1）：
4) 无CP大女主（事业/复仇/成长）
5) 现言脑洞（读心/反套路/黑化反制）
6) 宫斗宅斗/古言重生（智斗+逆袭）

你可以直接回编号，我会据此生成你的核心框架草案。
```
