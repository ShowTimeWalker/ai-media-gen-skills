# 去AI味轻度修订规范

## 适用范围

- 阶段五：`chapters/draft/卷X_序号_ch起始-ch结束.md`
- 阶段七：`release/release_version.md`、`release/synopsis.md`、`release/toc.md`

## 核心目标

在不改变剧情、事实和因果链的前提下，减少模板腔与机械表达，提升自然度和可读性。

## 轻度修订硬边界

### 禁止项（必须遵守）

- 禁止新增或删除事件
- 禁止改动角色动机和关系结论
- 禁止改动时间线顺序与关键设定
- 禁止改写伏笔锚点到不可追踪

### 允许项（可执行）

- 替换模板连接词（如“首先/其次/最后/总之”）
- 拆分过长机械句，调整停顿与节奏
- 合并重复表达，去除空泛总结句
- 在不改信息点前提下做句式多样化

## 常见 AI 味清洗清单

1. **流水账连接词过多**：减少“首先、其次、再次、最后”连续出现
2. **句式重复**：连续 3 句以上同构句时改写其中至少 1-2 句
3. **总结腔**：删除“综上所述、由此可见”等学术总结口吻
4. **解释过度**：删除读者已知信息的重复解释
5. **语气过平**：适度引入动作、感官、对话细节增强现场感

## 执行流程

1. 先锁定不可变信息（事件、设定、动机、时间线）
2. 进行轻度语言层改写（只动表达，不动事实）
3. 做一致性复核（与 `state/*`、章节蓝图、主线文件对齐）
4. 标记本次修订范围（写入 `state/change_log.md` 可选条目）

## 最小验收项

- 可读性：段落节奏自然，无明显模板化串联
- 自然度：高频套话明显下降，句式不机械重复
- 一致性：与 `framework.md`、`state/*`、章节事实无冲突
- 可追踪：关键伏笔和术语仍可在图谱中定位
