# 阶段三：主线与篇章故事

## 目标

构建全书主线与分卷剧情，保证每卷有独立目标并服务总主线。

## 输入

- `framework.md`
- `blueprint.md`

## 输出

- `story/story.md`
- `story/arcs/arcXX_<卷名>.md`

## 执行步骤

### 3A 主线确认

先生成全局主线：

- 主问题（主角最终要解决什么）
- 主对立（谁或什么阻碍主角）
- 主代价（主角必须付出的成本）
- 主成长（主角如何改变）

写入 `story/story.md`。

### 3B 分卷创作

按卷顺序串行生成剧情文件，必须遵循 `arc01 -> arc02 -> ...`，每卷至少包含：

- 卷目标
- 卷冲突升级点
- 卷内关键事件（起承转合）
- 卷末悬念/钩子
- 与总主线的连接点

每卷生成后，必须先执行卷内一致性自检（人物动机、冲突延续、规则边界），通过后再进入下一卷。

## 文件模板（卷文件）

```markdown
# ArcXX 卷名

## 卷定位
- 卷目标：
- 卷主题：

## 关键事件
1. ...
2. ...
3. ...

## 冲突升级
- 外部冲突：
- 内部冲突：

## 卷末钩子
- ...
```

## 门禁规则

- 未完成主线文件前，不生成章节蓝图。
- 未完成上一卷或上一卷自检未通过，不得开始下一卷。
- 全部卷按顺序完成后，才能进入阶段四。
