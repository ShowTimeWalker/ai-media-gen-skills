# 阶段五：正文生成（按批次）

## 目标

根据章节蓝图批量生成正文，保证风格稳定、信息推进清晰、章节衔接自然。

## 输入

- `framework.md`
- `chapters/outline/chXX.md`

## 输出

- `chapters/draft/chXX.md`

## 正文生成规则

- 必须严格遵守 `framework.md` 的视角和文风设定
- 章目标必须在正文内被完成或推进
- 章末必须保留明确钩子
- 避免重复叙事和无效回顾

## 批次策略

- 默认每批 3-5 章
- 每批完成后输出批次摘要：
  - 已完成章节
  - 关键推进点
  - 潜在一致性风险

## 章节文件模板

```markdown
# Chapter XX 章名

（正文内容）

---

## 章后元信息
- 本章目标完成度：
- 新增伏笔：
- 已回收伏笔：
- 下章接口：
```

## 门禁规则

每批生成后都要给出固定选项：

1. 继续下一批
2. 重写某一章
3. 回到阶段四调整蓝图
4. 进入阶段六一致性校对
