# 短视频创作规范

> 适用范围：团队协作标准化 & AI 批量内容生产
> 最后更新：2026-03-28

---

## 一、平台视频规格

### 1.1 各平台参数对照表

| 参数 | 抖音 | 快手 | 小红书 | 视频号 |
|------|------|------|--------|--------|
| **推荐比例** | 9:16 | 9:16 | **3:4**（首选） | 9:16 / 16:9 |
| **推荐分辨率** | 1080×1920 | 1080×1920 | 1080×1440 | 1080P+ |
| **时长上限** | ~15分钟 | 3~10分钟 | 15分钟 | 60分钟(手机)/8小时(PC) |
| **封面比例** | 9:16 | 9:16 | **3:4** | 9:16 / 16:9 |
| **格式** | MP4 | MP4 | MP4 | MP4 |

### 1.2 通用标准

- **分辨率**：1080P 及以上
- **格式**：MP4（H.264 编码）
- **帧率**：30fps（运动画面 60fps）
- **码率**：8-16 Mbps（VBR）

### 1.3 平台适配要点

- 小红书首页以 **3:4** 为主，上传 9:16 会被裁剪，**必须优先使用 3:4**
- 其他平台以 9:16 竖屏为流量主力
- 同一视频多平台分发时，至少导出两个版本：9:16（抖音/快手/视频号）+ 3:4（小红书）

## 二、内容结构与脚本规范

### 2.1 黄金四段式框架

```
钩子（0-3秒）→ 痛点/悬念（3-10秒）→ 核心内容/方案（10-45秒）→ 互动引导（结尾）
```

### 2.2 开头钩子模板

| 类型 | 模板 | 示例 |
|------|------|------|
| 痛点式 | 直击用户痛点 | "你是不是也遇到过……" |
| 悬念式 | 制造好奇心 | "我花了3个月才发现的真相……" |
| 反差式 | 打破认知 | "所有人都在做错的一件事" |
| 数据式 | 用数字冲击 | "99%的人不知道这个方法" |
| 情感式 | 直接触发情绪 | 温情/愤怒/感动开头 |
| 场景式 | 直接带入画面 | 第一人称视角开场 |
| 提问式 | 引发思考 | "为什么……？" |

### 2.3 平台风格适配

| 平台 | 风格定位 | 节奏 | 推荐时长 |
|------|---------|------|---------|
| 抖音 | 快准狠、强节奏 | 快 | 15-60s |
| 快手 | 接地气、真实感 | 中快 | 15-60s |
| 小红书 | 精致、治愈、种草 | 慢 | 30s-3min |
| 视频号 | 完播率优先、社交推荐 | 中 | 30s-3min |
| TikTok | 强视觉冲击 | 快 | 21-34s（黄金区间） |

### 2.4 脚本模板

每个脚本必须包含以下字段：

```
- 视频标题：
- 目标平台：
- 目标时长：
- 选题方向：
- 钩子（前3秒）：
- 主体内容（分镜描述）：
- 结尾互动引导：
- BGM/音乐需求：
- 字幕/文案要点：
- 封面文案：
```

## 三、拍摄与剪辑技术规范

### 3.1 拍摄参数标准

| 参数 | 标准值 | 说明 |
|------|--------|------|
| 分辨率 | 1080P（1080×1920） | 条件允许可拍 4K 用于裁切二次构图 |
| 帧率 | 30fps | 运动画面/慢动作用 60fps |
| 码率 | 8-16 Mbps（VBR） | 最低不低于 1024 Kbps |
| 编码 | H.264 | 兼容性最佳 |
| 色彩深度 | 8-bit（标准）/ 10-bit（专业） | - |
| 色彩采样 | 4:2:2（优先） | - |
| 音频 | AAC 48kHz 256Kbps 立体声 | - |

### 3.2 导出规格

#### 通用竖屏模板（抖音/快手/视频号）

| 参数 | 设置 |
|------|------|
| 分辨率 | 1080×1920 |
| 帧率 | 30fps |
| 编码 | H.264 |
| 码率 | VBR，目标 8-12 Mbps |
| 音频 | AAC 48kHz 256Kbps |
| 格式 | MP4 |

#### 小红书竖屏模板

| 参数 | 设置 |
|------|------|
| 分辨率 | 1080×1440 |
| 帧率 | 30fps |
| 编码 | H.264 |
| 码率 | VBR，目标 8-12 Mbps |
| 音频 | AAC 48kHz 256Kbps |
| 格式 | MP4 |

### 3.3 命名规范

**素材命名规则**：

```
YYYYMMDD_项目名_场景号_版本号
示例：20260328_产品测评_场景01_v1
```

**工程文件命名规则**：

```
YYYYMMDD_项目名_阶段_版本号
示例：20260328_产品测评_粗剪_v1
```

### 3.4 技术硬性要求

- 同一项目内分辨率、帧率、色彩空间**必须统一**，不得混用
- 画面不得出现黑边、拉伸变形
- 字幕不得超出安全区（画面边缘 10% 内）
- 音频峰值不超过 -1 dB，避免爆音
- 素材至少**双备份**（本地 + 云端）

## 四、团队协作流程

### 4.1 生产全流程

```
选题策划 → 脚本创作 → 拍摄执行 → 后期制作 → 审核交付 → 发布运营 → 数据复盘
```

### 4.2 各阶段职责

#### 阶段一：选题策划

- **输入**：热点追踪、竞品分析、用户反馈、历史数据复盘
- **输出**：选题库（周更，至少提前一周储备）
- **负责**：编导 + 运营
- **关键动作**：每周固定选题会，确定内容日历

#### 阶段二：脚本创作

- **输入**：确认选题
- **输出**：通过审核的分镜头脚本
- **负责**：策划/编剧
- **关键动作**：按 2.4 脚本模板填写，编导审核通过后方可拍摄

#### 阶段三：拍摄执行

- **输入**：审核通过的脚本
- **输出**：命名归档的原始素材
- **负责**：摄影师
- **关键动作**：严格按分镜拍摄，现场素材审看确认，素材当日备份

#### 阶段四：后期制作

- **输入**：原始素材 + 脚本
- **输出**：成片
- **负责**：剪辑师
- **流程**：素材整理 → 粗剪 → 精剪 → 配音/字幕 → 特效包装 → 成片

#### 阶段五：审核交付

- **输入**：成片
- **输出**：审核通过的最终版
- **负责**：编导/项目经理
- **三级审核**：
  1. **内容审核**：内容与策划方案是否一致，传播价值判断
  2. **画面审核**：画质、剪辑节奏、字幕校对、音频质量
  3. **发布审核**：合规性审查，规避敏感内容

#### 阶段六：发布运营

- **输入**：审核通过的成片 + 封面 + 标题 + 标签
- **输出**：各平台已发布内容
- **负责**：运营
- **关键动作**：按平台规格分别适配导出，定时发布

#### 阶段七：数据复盘

- **输入**：各平台数据
- **输出**：周报/月报 + 优化建议
- **负责**：运营
- **核心指标**：播放量、完播率、互动率、涨粉数、转化率
- **关键动作**：爆款经验沉淀复用，持续迭代内容模板

### 4.3 团队角色与考核

| 角色 | 核心职责 | 考核指标 |
|------|---------|---------|
| **编导/项目经理** | 统筹全局、选题策划、质量把控 | 内容质量评分、爆款率 |
| **策划/编剧** | 脚本撰写、文案创意、分镜设计 | 脚本产出量、通过率 |
| **摄影师** | 拍摄执行、灯光布景、素材采集 | 拍摄效率、素材合格率 |
| **剪辑师** | 视频剪辑、特效合成、调色输出 | 交付及时率、成片质量 |
| **运营** | 内容发布、数据分析、粉丝运营 | 播放量、涨粉数、转化率 |

### 4.4 协作工具

| 环节 | 推荐工具 |
|------|---------|
| 项目管理 | 飞书多维表格 / Notion |
| 脚本协作 | 腾讯文档 / 飞书文档 |
| 素材管理 | Eagle / 本地NAS / 云盘 |
| 视频审阅 | Frame.io / 剪映团队版 |
| 数据分析 | 蝉妈妈 / 飞瓜数据 / 新榜 |

## 五、AI 批量生产工作流

### 5.1 AI 工具矩阵

| 环节 | 工具 | 能力 |
|------|------|------|
| 脚本生成 | AI 脚本工具（ScriptGenerator 等） | 批量生成脚本，多版本 A/B 测试 |
| 图片素材 | 豆包（火山引擎）/ 橘子AI | 文生图、图生图 |
| 视频素材 | 橘子AI / Runway / Sora | 文生视频、图生视频 |
| 音乐/BGM | 天谱乐（Tianpuyue） | AI 作曲、歌曲、歌词 |
| 批量混剪 | MoneyPrinterTurbo / AI短视频工厂 | 模板化自动剪辑 |
| 文件交付 | 七牛对象存储 | 上传获取公网/签名链接 |
| 整体编排 | content_generation_workflow | 统一调度各 AI 能力 |

### 5.2 批量生产标准化流程

```
1. 选题库批量生成
   └── AI 辅助热点分析 → 批量生成选题 → 人工筛选确认

2. 脚本批量生成
   └── 同一主题生成多版本脚本 → A/B 测试选优

3. 素材批量生成
   └── AI 生图 → AI 生视频 → AI 生音乐 → 素材归档

4. 批量合成
   └── 模板化自动剪辑 → 批量混剪输出

5. 多平台适配
   └── 按平台规格自动转码（9:16 + 3:4）→ 封面适配

6. 批量发布
   └── 标题/标签/封面批量配置 → 定时分发

7. 数据回流
   └── 自动采集各平台数据 → 分析反馈 → 反哺选题库
```

### 5.3 AI 生成内容规范

- AI 生成的视频/图片/音乐需**质量审核**后方可使用
- AI 脚本需**人工审核**确认逻辑通顺、信息准确
- 注意各平台对 **AI 生成内容的标注要求**
- AI 生成素材同样遵循命名规范和双备份要求
- 批量生产时，每批次保留**模板参数记录**，便于复用
