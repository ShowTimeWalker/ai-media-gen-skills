# AGENTS.md

## 沟通规则

1. 始终使用中文与用户交流。
2. 回复应尽量直接、明确，优先给出可执行结果，不做无关展开。

## 项目定位

1. 本项目用于测试各类 API，以及搭建和调试 skill。
2. skill 可通过 Python 或 Node.js 实现，相关开发和测试都应围绕这一目标展开。

## 目录约定

1. Python 测试脚本放在 `scripts/python/`。
2. Node.js 测试脚本放在 `scripts/node/`。
3. 外部参考文档放在 `docs/` 目录，涉及方案、接口或实现细节时优先查阅该目录。
4. 通用资源、模板、示例素材等放在 `resources/` 目录，编写脚本、调试 skill 或整理测试输入时优先复用该目录内容。
5. `api_key/` 目录下的接口凭据或模型配置文件统一使用 JSON 格式，你可以在写入/测试脚本的时候使用。

## 依赖与环境

1. Python 相关操作优先使用 `uv`。
2. Node.js 相关操作优先使用 `pnpm`。
3. 执行脚本、安装依赖或运行测试时，优先使用本项目自己的虚拟环境或项目环境，不要默认使用全局环境。
4. 执行 Python 命令时必须使用 `uv run`，不要直接调用系统 `python`。
5. 如果 Python 脚本运行或测试缺少依赖，优先使用 `uv add` 管理项目依赖。

## Skill 创建约定

1. 如果用户要求创建 skill，默认按本项目约定完成，不需要额外切换风格。
2. skill 的 `name` 使用英文。
3. 其他可见内容尽量使用中文，包括说明文本、注释、提示词和脚本内说明。
4. 注意你在创建skill时严禁直接引用字面量

## 执行原则

1. 修改文件或新增脚本时，保持目录结构清晰，命名与用途一致。
2. 若用户需求与以上约定冲突，优先遵循用户的明确要求。
3. 新增或重命名文件时，文件名优先使用英文。
